我觉得下一步最值得改的，不是再“多找票”，而是把**放行后的执行质量**继续做厚。今天系统已经能给主票了，下一层挑战就是：主票什么时候买、买多少、错了怎么退、第二天怎么复盘。

**最优先的改进**

1. **补真正的个股盈亏比字段**

   现在报告里已经能写胜率，但“盈亏比”还主要靠 `next_close_return_mean`、`next_open_to_close_return_mean` 来解释收益质量。下一步最好在 artifact 里直接输出：

   - 平均盈利
   - 平均亏损绝对值
   - payoff ratio = 平均盈利 / 平均亏损
   - profit factor = 总盈利 / 总亏损
   - expectancy = 胜率 × 平均盈利 - 亏损率 × 平均亏损

   这样就能避免 `002463 沪电股份` 这种“胜率高但均值差”的误读。

2. **主票放行要有“最低执行阈值”**

   今天 `300054 鼎龙股份` 作为主票是合理的，但还可以加一个执行门槛，比如：

   - 高开超过多少不追
   - 竞价量低于多少不做
   - 开盘 5 分钟不能站稳某个价位就取消
   - 回踩不破 VWAP / 分时均线才确认

   现在报告说“等 confirmation”，但 confirmation 还偏文字化。可以把它变成结构化规则。

3. **区分“冲高型”和“收盘延续型”**

   BTST 最关键的问题是：到底是吃次日冲高，还是持到收盘。

   现在已经有：

   - `next_high>=2% 命中率`
   - `next_close 正收益率`
   - `next_open_to_close_return_mean`

   下一步可以直接给标签：

   - `intraday_spike_only`
   - `close_continuation`
   - `gap_risk_high`
   - `hold_allowed`
   - `must_trim_on_spike`

   比如 `600183 生益科技` 就应该明确是 intraday-only，不应写进隔夜持有候选。

4. **增加仓位建议，不只是名单**

   现在是“谁能做”，还缺“做多大”。

   可以按证据强度给仓位等级：

   - A 类：主票，0.5x-1.0x 标准仓
   - B 类：备选，0.3x-0.5x
   - C 类：near-miss，只观察或 0.2x 试错
   - D 类：shadow / research-only，不交易

   这样执行清单会更像真实交易计划。

5. **加入失败条件**

   报告现在对“做什么”说得比较清楚，但对“什么情况下不做”还可以更硬。

   例如对 `300054 鼎龙股份`：

   - 若竞价高开过大且开盘 3 分钟放量下杀，不做
   - 若开盘后不能维持强于板块，不做
   - 若同题材核心股走弱，不做
   - 若第一次突破失败后回抽无量，不做第二次追

   好交易系统一定要知道什么时候闭嘴。

6. **把规则版和 LLM 版做成冲突诊断**

   今天规则版很积极，LLM 版只放 2 只。这种差异很有价值，应该显式输出：

   - 规则强、LLM 强：主攻
   - 规则强、LLM 弱：只观察
   - 规则弱、LLM 强：谨慎试错
   - 规则弱、LLM 弱：空仓

   现在是两份文档并列，下一步可以做一个“交叉裁决表”。

7. **复盘闭环要自动化**

   第二天收盘后，系统应该自动问：

   - 主票是否触发入场？
   - 入场后最大浮盈是多少？
   - 是否达到 next_high>=2%？
   - 收盘是否为正？
   - 如果亏损，是买点错、选股错、还是市场错？
   - near-miss 里有没有比主票更好？如果有，为什么没放行？

   这会让优化从“调参数”变成“学错因”。

**我会优先做的三件事**

第一，补 `payoff_ratio / profit_factor / expectancy`，把胜率和赔率分开。  
第二，把 `confirmation` 结构化成明确开盘条件。  
第三，做一张规则版 × LLM 版的交叉裁决表，直接告诉你哪些是真主线，哪些只是热闹。

一句话：现在系统已经从“能筛”走到“能放行”，下一步要让它从“能放行”走到“能执行、能复盘、能知道自己错在哪里”。
