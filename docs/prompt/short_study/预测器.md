# 从第一性原理出发，我们为了获取更大的收益，你还有更好的方案吗？

你这个追问逼我从零想。前面三轮我都在"资金面塞进现有架构的哪个位置"上打转 —— 换 composite、加 alpha_score、建第二个系统。这些都是修补，不是第一性原理。

  让我真的从第一性出发重推。

  第一性：我们到底在优化什么？

  目标：最大化 T+5/T+10 组合收益。

  收益 = Σ (仓位ᵢ × 个股收益ᵢ)

  要最大化它，我需要知道每只票未来 5-10 天能涨多少，然后：
  1. 选涨幅最高的几只
  2. 涨幅高 + 把握大的，仓位重一点（Kelly）

  这是一个预测问题 —— 预测每只票的 T+5/T+10 收益率。

  深层洞察：现有系统根本不是预测器

  ┌─────────────────────────┬──────────────────────┬─────────────────────┐
  │          系统           │     它在回答什么     │     它输出什么      │
  ├─────────────────────────┼──────────────────────┼─────────────────────┤
  │ composite_score         │ "这票质地好不好？"   │ 0-1 质地分          │
  ├─────────────────────────┼──────────────────────┼─────────────────────┤
  │ BUY gate                │ "这票安不安全？"     │ BUY/HOLD/AVOID 标签 │
  ├─────────────────────────┼──────────────────────┼─────────────────────┤
  │ alpha_score（我上一稿） │ "这票资金面强不强？" │ 0-1 信号分          │
  └─────────────────────────┴──────────────────────┴─────────────────────┘

  这些全是"打分/分类器"，没有一个是"收益预测器"。

  打分器的致命缺陷：
  - 不校准收益 —— composite=0.7 不代表涨 7%，它只是"质地好"
  - 池内无排序力 —— 这就是你 R6 + 单调性倒挂的根因，打分器不是为预测收益设计的
  - 无法指导仓位 —— 0.7 vs 0.5 该买几股？打分器答不出，预测器能（预测收益 × 置信度 → Kelly 仓位）
  - 不能发现因子交互 —— 主力流入 + 涨停板 + 龙虎榜 = 催化剂共振，这种交互打分器抓不到

  "质地好"≠"会涨"。 高质地票没催化剂可以横盘 30 天；低质地票有催化剂可以 5 天涨 30%。我前三稿（含两系统）都在打分器框架里打转，注定天花板低。

  第一性原理方案：构建收益预测器（不是打分器）

  预测器目标: 对每只票输出 predicted_T5_return = +5.2% (CI 2%-8%)

  预测器形式 (IC 加权线性多因子模型, 行业标准):
    predicted_T5 =
        IC_trend        × trend_zscore        +
        IC_capital_flow × capital_flow_zscore +
        IC_news         × news_zscore         +
        IC_sentiment    × sentiment_zscore    +
        IC_fundamental  × fundamental_zscore  +
        IC_MR           × MR_zscore           + α

    每个 IC = 该因子对 T+5 收益的历史信息系数 (走前 IC, regime 分层)

  这跟我上一稿"两系统"的本质区别：

  ┌────────────────────┬─────────────────────────────────────┬───────────────────────────────────────────────┐

  为什么"收益预测器"统一了防守和进攻

  打分器框架下，防守和进攻是两个目标、两个系统。预测器框架下，它们是同一个预测的不同阈值：

  ┌───────────────────────────────────┬────────────────┬──────────────────┐
  │             预测收益              │      含义      │       动作       │
  ├───────────────────────────────────┼────────────────┼──────────────────┤
  │ predicted_T5 ≥ +5%, CI 下界 > +2% │ 高把握高收益   │ 重仓 BUY（进攻） │
  ├───────────────────────────────────┼────────────────┼──────────────────┤
  │ predicted_T5 ≥ +2%, CI 跨 0       │ 正期望但不确定 │ 轻仓 BUY（中性） │
  ├───────────────────────────────────┼────────────────┼──────────────────┤
  │ predicted_T5 ≈ 0                  │ 无 edge        │ HOLD（防守）     │
  ├───────────────────────────────────┼────────────────┼──────────────────┤
  │ predicted_T5 < 0                  │ 负期望         │ AVOID（防守）    │
  └───────────────────────────────────┴────────────────┴──────────────────┘

  一个模型，四个动作，按预测收益和置信度自然切分。 不需要"两个系统"，不需要人工定义"什么时候进攻什么时候防守" —— 预测器自己回答。

  为什么这真正更好（第一性）

  1. 目标对齐：预测器直接优化 E(收益)，打分器优化"质地分"。你想要收益，就该预测收益。
  2. 数据驱动权重：IC 高的因子自动权重大。不用人工辩"资金面该占 20% 还是 30%" —— 历史数据说了算。
  3. 可发现因子无效：如果某因子 IC≈0（比如 crisis 下 trend IC 可能转负），预测器自动把它权重压到 0。打分器做不到 —— 它会一直给 trend 30% 权重，即使在 crisis。
  4. 直接解决 R6/单调性倒挂：IC 加权是为单调性设计的（IC > 0 → 高分对应高收益）。如果加权后仍倒挂，说明因子集需要换，这本身就是诊断信号。
  5. 可叠加 ML：线性 IC 加权是起点。如果欠拟合，第二阶段上 XGBoost/LightGBM 捕捉非线性 + 交互（主力+涨停+龙虎榜共振）。
  6. 行业正道：Numerai / WorldQuant / 明汯都是这套。你记忆里的 [[project_btst_ic_analysis]] 已经做过单因子 IC 分析 —— 这正是基础设施。

  这是第一性原理答案：不要建"打分系统"（composite / alpha_score / 两系统），要建"收益预测器"。 你前两轮的痛（池内无排序、危机 0
  BUY、打分≠收益）的根因是系统在打分而非预测。预测器框架下这些痛自然消解。

  认同这个方向吗？认同的话下一步是 Phase 0 的 IC 普查设计 —— 具体哪些因子、哪些 regime 切片、走前窗口。
