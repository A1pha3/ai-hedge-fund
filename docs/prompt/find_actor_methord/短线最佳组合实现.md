## 短线起爆点捕捉策略：压缩-爆发-突破 三因子系统

### 1. 策略的第一性原理

市场价格短期的剧烈运动，本质是由**波动率从极低状态向极高水平回归**的过程。在微观结构上，它通常经历三个阶段：

1. **沉寂（压缩）**：市场陷入窄幅盘整，多空力量暂时均衡，波动率被压缩至低位。这像一个被压紧的弹簧，积蓄了巨大的势能。
2. **引燃（爆发）**：由新信息或大资金驱动，成交量突然急剧放大，市场能量瞬间释放，打破均衡。
3. **破位（方向确认）**：价格果断突破前期盘整边界，触发大量止损单和追单盘，形成短期趋势的自我强化。

我们的三因子系统正是直接捕获这三个物理过程：

- **波动率压缩因子** —— 识别“弹簧压紧”
- **能量爆发因子** —— 确认“推力足够”
- **方向突破因子** —— 执行“起爆入场”

这套逻辑的优势在于：我们交易的不是价格的**当前位置**，而是能量从积蓄到爆发的**瞬时过程**。因此入场更及时，止损更窄，盈亏比天然更大。

### 2. 因子定义与计算

下面使用标准 OHLCV 数据（开盘、最高、最低、收盘、成交量）。所有参数均附建议默认值，可后续通过回测优化。

#### 2.1 波动率压缩因子（Volatility Squeeze）

**原理**：当市场波动率处于近期极低水平时，未来波动率扩张的概率和力度都显著增加。

**计算**：使用布林带宽度（Bollinger Band Width，BBW）或平均真实波幅（ATR）的近期分位数。

**方案 A：布林带宽度百分位（推荐）**
- 计算 20 周期布林带（中轨 MA20，上下轨 MA20 ± 2倍标准差）
- 布林带宽度 `BBW = (上轨 - 下轨) / 中轨`
- 计算 BBW 在过去 N 个周期的历史百分位 `BBW_pct = (当前BBW - N周期最低BBW) / (N周期最高BBW - N周期最低BBW)`
- 建议参数：`N_lookback = 100`（用于百分位计算），压缩阈值 `< 0.2`（即 BBW 处于历史最低的 20% 区间内）

**方案 B：ATR 低位（备选，更简洁）**
- 计算 14 周期 ATR
- 计算 ATR 的 20 周期最低值 `ATR_lowest_20`，以及当前 ATR 与该最低值的比值 `ATR_ratio = ATR / ATR_lowest_20`
- 当 `ATR_ratio < 1.1`（即 ATR 接近其 20 周期最低点）时，视为压缩。此方法更健壮，能适应波动率的长期变化。

**压缩因子信号输出**：满足条件时，`Squeeze = True`；否则 `False`。

#### 2.2 能量爆发因子（Volume Surge）

**原理**：真正的突破必然伴随着成交量的显著放大。成交量是衡量市场参与者“急切程度”的最直接指标。

**计算**：成交量相对于近期均值的放大倍数。

- 计算成交量的 M 周期简单移动平均 `Vol_MA = SMA(Volume, M)`
- 定义当前成交量比率 `Vol_Ratio = Volume / Vol_MA`
- 爆发阈值：`Vol_Ratio > 1.5` 或 `2.0`（1.5 更敏感，2.0 更严格）。建议初始使用 `1.5`，以提高信号数量进行回测筛选。

**更稳健的版本**：要求过去 K 根 K 线中至少有 J 根的 `Vol_Ratio > 阈值`，避免单一 tick 的脉冲噪音。短线可直接使用瞬时判断。

**爆发因子信号输出**：满足时 `Surge = True`。

#### 2.3 方向突破因子（Breakout）

**原理**：价格超越近期波动区间的边界，确认压缩区已被打破，趋势方向确立。

**计算**：使用唐奇安通道（Donchian Channel）的突破，但周期与波动率压缩窗口适配。

- 确定盘整区间高点：`HH = 过去 P 周期的最高价最大值`
- 确定盘整区间低点：`LL = 过去 P 周期的最低价最小值`
- 建议参数：`P = 20` 或 `30`。这个周期应与波动率压缩的观察窗口匹配（如压缩是用 100 周期百分位，突破可用较短的 20 周期，因为突破的正是近期的窄幅盘整）。

**突破信号**：
- 向上突破：`Close > HH`（或 `High > HH`，但用收盘价更稳定，减少毛刺）
- 向下突破：`Close < LL`

**方向确认信号**：向上突破时 `Breakout_Up = True`，向下突破时 `Breakout_Down = True`。

### 3. 开仓规则（综合三因子共振）

只有当三个因子在**同一根 K 线**或非常紧密的窗口内同时满足，才触发开仓。这样可以大幅度过滤掉低质量信号。

#### 3.1 做多开仓条件（Long Entry）
以下条件必须**同时**成立：
1. **波动率压缩**：当前或上一根 K 线的压缩因子 `Squeeze = True`（允许延迟 1 根，避免错过）。
2. **能量爆发**：当前 K 线的 `Surge = True`（成交量显著放大）。
3. **方向突破**：当前 K 线收盘价向上突破 P 周期高点 `HH`。
4. **趋势过滤（可选但推荐）**：价格在 50 周期均线上方，或 50 均线斜率向上。这能避免在长期下跌趋势中做多反弹，提高胜率。短线可不加，但建议保留。

#### 3.2 做空开仓条件（Short Entry）
条件对称：
1. `Squeeze = True`
2. `Surge = True`
3. `Close < P 周期低点 LL`
4. （可选）价格在 50 均线下方。

#### 3.3 开仓执行细节

- **入场价格**：收盘价突破时，用下一根 K 线的开盘价入场（以消除收盘价突破后次根瞬间回拉的影响），或者用当前收盘价入场（更激进）。建议使用 **下一根开盘价**，更接近实盘可行性。
- **初始止损**：必须紧贴压缩区间的对立边界。
  - 做多：初始止损设在 `LL` 下方一点（如 `LL - 0.5 * ATR`）。
  - 做空：初始止损设在 `HH` 上方一点（如 `HH + 0.5 * ATR`）。
  这保证了我们的风险非常有限，而盈利空间是向突破方向打开的，天然高盈亏比。
- **仓位计算**：基于固定金额止损或固定百分比风险。例如，单笔亏损不超过总资金的 1%。止损点数已知（入场价与止损价之差），倒推仓位大小。

### 4. 持仓管理和出场规则（简要）

系统核心在开仓，但出场规则决定了利润能否落袋。建议结合你已有的“位置因子+均线距离”框架，或者采用以下简单而有效的出场：

- **移动止损**：开仓后，将止损移动到盈亏平衡点（入场价）之后，可启用追踪止损。追踪止损可以用：
  - 均线（如 20 周期 EMA），当价格反向穿越均线时出场。
  - 超级趋势指标（SuperTrend）。
  - 近期低点/高点（如 10 周期最低价做多止损）。
- **时间止损**：若开仓后 N 根 K 线（如 5 根）仍无利润且未触及止损，主动平仓。短线能量爆发如果没有立即延续，则信号可能失败。

### 5. 伪代码实现（供 Agent 直接翻译）

以下是 Python 风格伪代码，清晰展示逻辑。

```python
# 参数设置
BB_PERIOD = 20          # 布林带周期
BB_STD = 2.0
LOOKBACK_PCT = 100      # BBW 百分位回顾期
SQUEEZE_THRESHOLD = 0.2 # 压缩阈值

VOL_MA_PERIOD = 20      # 成交量均线周期
VOL_RATIO_THRESHOLD = 1.5 # 放量倍数

DONCHIAN_PERIOD = 20    # 突破区间周期

TREND_MA_PERIOD = 50    # 趋势过滤均线

# 计算因子
def calc_indicators(df):
    # 布林带宽度
    df['ma'] = df['close'].rolling(BB_PERIOD).mean()
    df['std'] = df['close'].rolling(BB_PERIOD).std()
    df['bb_upper'] = df['ma'] + BB_STD * df['std']
    df['bb_lower'] = df['ma'] - BB_STD * df['std']
    df['bbw'] = (df['bb_upper'] - df['bb_lower']) / df['ma']
    # BBW 百分位
    df['bbw_low'] = df['bbw'].rolling(LOOKBACK_PCT).min()
    df['bbw_high'] = df['bbw'].rolling(LOOKBACK_PCT).max()
    df['bbw_pct'] = (df['bbw'] - df['bbw_low']) / (df['bbw_high'] - df['bbw_low'])
    df['squeeze'] = df['bbw_pct'] < SQUEEZE_THRESHOLD

    # 成交量
    df['vol_ma'] = df['volume'].rolling(VOL_MA_PERIOD).mean()
    df['vol_ratio'] = df['volume'] / df['vol_ma']
    df['surge'] = df['vol_ratio'] > VOL_RATIO_THRESHOLD

    # 唐奇安通道
    df['hh'] = df['high'].rolling(DONCHIAN_PERIOD).max()
    df['ll'] = df['low'].rolling(DONCHIAN_PERIOD).min()

    # 趋势过滤
    df['trend_ma'] = df['close'].rolling(TREND_MA_PERIOD).mean()
    df['is_uptrend'] = df['close'] > df['trend_ma']
    df['is_downtrend'] = df['close'] < df['trend_ma']

    return df

# 生成信号
def generate_signals(df):
    df = calc_indicators(df)
    # 做多条件
    long_condition = (
        df['squeeze'].shift(1) | df['squeeze']  # 压缩信号可持续1根K线
    ) & (
        df['surge']
    ) & (
        df['close'] > df['hh'].shift(1)  # 当前收盘价突破前一根的HH（更稳妥：突破上根HH）
    ) & (
        df['is_uptrend']  # 可选
    )
    # 做空条件
    short_condition = (
        df['squeeze'].shift(1) | df['squeeze']
    ) & (
        df['surge']
    ) & (
        df['close'] < df['ll'].shift(1)
    ) & (
        df['is_downtrend']  # 可选
    )
    df['long_signal'] = long_condition
    df['short_signal'] = short_condition
    return df
```

### 6. 回测与优化指南

1. **时间周期**：建议在 **5 分钟、15 分钟、30 分钟** K 线上测试。这个系统在日内短线流动性最大的品种（如股指期货、主流外汇、活跃加密货币）上表现最佳。
2. **优化参数顺序**：
   - 先固定 `DONCHIAN_PERIOD`（20），调整 `VOL_RATIO_THRESHOLD` 在 1.2~2.5 之间寻找最优胜率/盈亏比平衡。
   - 再调整 `SQUEEZE_THRESHOLD`（0.1~0.3）。
   - 最后微调 `LOOKBACK_PCT`（100~250）。
3. **避免过拟合**：采用 Walk-forward Analysis（向前推进分析）或交叉验证，切勿在所有历史数据上一次性优化所有参数。

### 7. 为什么这个系统在逻辑上难以被击败

- 它没有预测，只跟随能量。一旦市场没有“压缩→爆发”这个物理过程，系统就安静等待。
- 它的盈亏比由市场结构决定：盘整区间宽度 vs 趋势释放长度。在短线，这种结构盈亏比常常可达 2:1 到 5:1。
- 它的胜率由“成交量筛选”保证：随机假突破往往不放量，有准备的资金动作才放量。

**下一个动作**：让你的 agent 先在 6-12 个月的样本外数据上运行，只记录信号不开仓，亲眼观察信号质量和盈亏分布。这会让你对这套逻辑建立起超越回测曲线的信心。
