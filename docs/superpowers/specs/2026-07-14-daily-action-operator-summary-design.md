# Daily Action 操作员摘要设计

日期：2026-07-14

## 目标

让 `uv run python src/main.py --daily-action` 的操作者无需理解内部错误码，就能立即分辨：

1. 系统正常但今日没有合格信号；
2. 数据或交易日护栏阻止了新计划；
3. 已成功生成次日计划；
4. 应执行的下一步操作。

不改变选股、账本、成交、退出或 manifest 判定逻辑，只改操作员文本渲染。

## 输出结构

输出固定按以下顺序组织：

1. 标题：`每日动作 v2（模拟台账）`
2. 操作员摘要：结论、原因、影响、建议
3. 参考价计划
4. 模拟成交
5. 确认成交
6. 退出挑战者
7. 不可计划候选及生命周期明细
8. 技术诊断

所有空栏目明确显示 `无`，不再留下只有标题的空白段落。

## 摘要状态

状态按以下优先级计算：

1. 存在新计划或同日持久化计划：`✅ 当前有 N 个次日买入计划`。若同时存在警告，计划继续有效，摘要另列安全降级信息。
2. 没有计划且存在阻断原因：`⛔ 今日未生成新的次日买入计划`，说明安全护栏已阻止新计划；已有持仓仍按既定规则管理。
3. 没有计划且没有阻断原因：`ℹ️ 今日无符合条件的次日买入信号（系统运行正常）`。

`regime_authorization_evidence_unavailable` 属于非阻断警告：它只把 crisis/risk-off 单票上限安全降为 10%，不能据此声称“没有计划”。其余已知 manifest/交易日原因属于阻断原因。未知原因按阻断处理，保持 fail-closed。

摘要不得把“无信号”和“系统阻断”合并成同一种结果。

## 阻断原因映射

面向操作者显示中文解释，同时保留原始代码供审计：

| 内部代码 | 中文原因 | 建议 |
|---|---|---|
| `healthy_manifest_missing` | 缺少当前交易日的健康数据清单 | 收盘后先运行 `uv run python src/main.py --auto`，成功后重跑 daily-action |
| `manifest_identity_mismatch` | 数据清单与当前交易日或运行批次不匹配 | 重新运行 `--auto`，不要混用旧报告与新缓存 |
| `manifest_invalid` | 数据清单结构不完整或无法验证 | 重新运行 `--auto`；若仍出现，检查 auto 输出中的数据质量错误 |
| `calendar_unavailable` | 权威交易日历不足，无法确定精确入场日或持有期 | 更新交易日历/缓存后重跑，禁止猜测自然日 |
| `regime_authorization_evidence_unavailable` | 市场状态加仓缺少可验证授权证据 | 计划仍按单票 10% 安全上限执行，无需手工放大仓位 |

未知代码显示为 `系统安全护栏触发`，并在技术诊断中保留原值，避免因新增代码而丢失信息。

多个原因按首次出现顺序逐条显示，不展示拼接后的重复 `block_reason` 与 `block_reasons` 两行。

## 示例

```text
每日动作 v2（模拟台账）

结论：⛔ 今日未生成新的次日买入计划
原因：缺少当前交易日的健康数据清单
影响：不会创建新买入计划；已有持仓仍按既定规则管理
建议：
  1. 收盘后先运行：uv run python src/main.py --auto
  2. 成功后重新运行：uv run python src/main.py --daily-action

参考价计划：无
模拟成交：无
确认成交：无
退出挑战者：无（SHADOW ONLY，不触发交易）

技术诊断：
  block_reasons=healthy_manifest_missing
```

## 兼容性与安全

- `render_daily_action_v2` 增加仅控制展示的关键字参数 `verbose=False`；dispatcher 以 `"--verbose" in argv` 显式传入。业务运行对象和服务接口不增加展示状态。
- `--verbose` 显示技术诊断、逐股票 manifest 阻断详情和原始代码。
- 非 verbose 模式仍显示中文结论、原因、影响与建议，但可省略逐股票技术细节。
- `SHADOW ONLY` 标识必须保留，不能让退出挑战者看起来像真实退出指令。
- 不根据错误码推测数据内容；所有建议只描述安全恢复步骤。
- 不修改 JSON、SQLite 或其他机器接口，避免破坏下游消费者。

## 测试标准

新增渲染回归测试覆盖：

1. `healthy_manifest_missing` 显示中文原因、两步建议和原始诊断码；
2. 多个阻断原因按顺序显示且不重复输出旧的单数/复数字段；
3. 系统健康但无信号时明确显示“系统运行正常”；
4. 有计划时显示准确数量；
5. `regime_authorization_evidence_unavailable` 显示为安全降级警告且不掩盖已有计划；
6. 四个主要空栏目均显示 `无`；
7. 已有持仓但新计划被阻断时，摘要明确说明已有持仓继续管理；
8. 未知错误码安全降级为通用中文解释并保留原值；
9. 现有成交来源标签、退出挑战者 `SHADOW ONLY` 语义不变。
