# 2026-03-18 Re-entry 质量专项

## 1. 结论摘要

P1-1 的核心结论已经可以明确：当前窗口里的 re-entry 主要不是在修复机会损失，而是在不同程度上放大 churn；其中 `300724` 是唯一真正的 re-entry 主样本，`603993` 并不构成 re-entry 问题主轴。

更具体地说：

1. `300724` 的回补在 `exit_fix`、`cooldown`、`trading_days`、`retrace6`、`prod_validation` 五组产物中都能观察到，但没有一组能证明回补后带来了更高质量收益。
2. `cooldown` 与 `trading_days` 的作用更像“延后和减少坏回补”，而不是把坏回补变成好回补。
3. `prod_validation` 的 `20260226` 回补是最典型的低质量 re-entry：它突破了 guardrail，却没有带来更优的总结果。
4. `603993` 在这些 runs 里基本是一次建仓、长期持有、再由 `time_stop` 或 `profit_retrace` 退出，不是 churn 的主要来源。

因此，下一阶段不应把目标表述成“让 300724 更容易重新买回来”，而应表述成“只允许极少数经过更强确认的回补继续存在”。

## 3. 300724 的 re-entry 序列

### 3.1 `exit_fix`：最典型的 churn 放大版本

`300724` 的真实交易序列：

1. `20260204` 买入 `100`
2. `20260205` 卖出 `100`
3. `20260206` 再买入 `100`，距前次卖出仅 `1` 天
4. `20260213` 再卖出 `100`
5. `20260225` 再买入 `100`，距前次卖出 `12` 天

这个版本的问题非常清楚：

1. 第一轮回补几乎是次日回补，根本没有形成新的质量确认。
2. 第二轮回补虽然间隔拉长到 `12` 天，但最终仍没有证明自己是高质量再入场。
3. 该 run 的总收益为 `-3.9769%`，是对比组里较差的一档，说明频繁回补没有换来更好的结果。

### 3.2 `cooldown`：延后坏回补，但没有把它变成好回补

`300724` 的序列收敛为：

1. `20260204` 买入 `100`
2. `20260205` 因 `hard_stop_loss` 卖出 `100`
3. `20260213` 再买入 `100`，距前次卖出 `8` 天

这里的改善主要体现在：

1. 从两次回补缩到一次回补。
2. 把最差的 `1` 天回补切掉了。

但它仍不足以证明回补质量提升，因为：

1. `cooldown` 的总收益仍为 `-2.9036%`。
2. 回补后也没有带来稳定的替代收益来源。

所以 `cooldown` 更像是在抑制 churn，不是在制造高质量回补。

### 3.3 `trading_days` 与 `retrace6`：最像“可接受”的版本，但仍不是高质量回补

这两个版本的 `300724` 序列高度一致：

1. `20260204` 买入 `100`
2. `20260205` 因 `hard_stop_loss` 卖出 `100`
3. `20260225` 再买入 `100`，距卖出 `20` 天
4. `20260304` 再次因 `hard_stop_loss` 卖出 `100`

它们比前两组更好，主要是因为：

1. 回补间隔被明显拉长。
2. 回补次数被压到一次。
3. 组合总结果改善到 `-2.4428%` 与 `-2.3152%`。

但这仍然不等于高质量回补，原因在于：

1. 这次回补最终还是以 `hard_stop_loss` 结束。
2. 没有出现“回补后延续上涨并改善收益质量”的证据。
3. 最多只能说明“这个版本比更激进的回补方式伤害更小”。

因此，`trading_days` / `retrace6` 的正确描述应是：

- 它们实现了更低 churn 的回补
- 但还没有实现真正高质量的回补

### 3.4 `prod_validation`：明确应该被拦下的低质量回补

`300724` 的序列是：

1. `20260206` 买入 `100`
2. `20260213` 因 `hard_stop_loss` 卖出 `100`
3. `20260226` 再买入 `100`，距卖出 `13` 天

这次回补的问题不是间隔太短，而是：

1. 它正好落在 `20260226 / 300724` 的 guardrail 样本上。
2. `paper-trading-reentry-validation-20260317.md` 已确认当日 `score_final = 0.2250`，属于应该被更高确认门槛拦下的边缘回补。
3. 尽管 `prod_validation` 的账面收益比多数 live runs 好，但它仍然明显差于 `baseline`：
   - `prod_validation`: `-1.2525%`
   - `baseline`: `-0.5520%`

这说明这次回补没有修复机会损失，反而带来了额外劣化。

### 3.5 `baseline`：当前最合理的处理方式是“不回补”

`baseline` 下的 `300724` 序列只有：

1. `20260206` 买入 `100`
2. `20260213` 因 `hard_stop_loss` 卖出 `100`
3. `20260226` 不再回补

这是当前唯一被证据直接支持的默认口径，因为：

1. 它避免了 `20260226` 的低质量再入场。
2. 它的总结果优于 `prod_validation`。
3. 它与 benchmark guardrail 完全一致。

## 4. 603993 的角色

`603993` 在 P1-1 里最大的意义不是 re-entry，而是作为对照组证明：当前 churn 问题并不是所有票都有。

各组序列基本一致：

1. `20260203` 买入 `200`
2. `20260204` 再买入 `100`
3. 之后长期持有
4. 最终由 `time_stop`、`profit_retrace` 或窗口结束处理退出

关键结论：

1. `603993` 没有出现“卖出后再次买回”的 re-entry 现象。
2. 它的问题更像“低弹性静态持有”，而不是“弱仓位反复回补”。
3. 因此在 P1-1 里，它主要承担参照作用：说明当前 re-entry 质量问题是集中在 `300724` 上，而不是整个组合普遍重进。

## 5. 重进前后收益质量判断

按当前证据，可以把 re-entry 分成三类：

### 5.1 明确低质量回补

1. `exit_fix` 中 `20260206` 的 `300724` 次日回补。
2. `prod_validation` 中 `20260226` 的 `300724` guardrail 回补。

共同特征：

1. 没有更强确认。
2. 回补后没有证明收益质量改善。
3. 其中至少一例直接让整体结果劣于不回补版本。

### 5.2 低 churn 但仍未证优的回补

1. `cooldown` 的 `20260213` 回补。
2. `trading_days` / `retrace6` 的 `20260225` 回补。

共同特征：

1. 间隔更长，次数更少。
2. 比激进回补伤害小。
3. 但最终仍没有形成“回补后改善收益质量”的硬证据。

### 5.3 当前唯一被验证的优先策略

1. `baseline` 的“不回补”。

在现有窗口里，它比 `prod_validation` 更优，也没有破坏 benchmark guardrail。

## 6. 最终判断

P1-1 当前可以先收口为四条：

1. `300724` 的 re-entry 目前没有任何一条被证明是高质量回补。
2. `cooldown`、`trading_days`、`retrace6` 的价值在于减少低质量回补，而不是证明回补本身有效。
3. `prod_validation` 的 `20260226` 回补是明确的反例，它证明“重新买回来”并不自动等于修复机会损失。
4. `603993` 不是 re-entry 问题主轴，下一步如果继续拆它，应放到 lifecycle / 持有质量，而不是回补质量。

## 7. 对后续任务的指向

1. P1-2 `300724 lifecycle` 应进一步拆 `20260204/05/06` 与 `20260225/20260304` 两段路径，回答为什么较晚回补仍然会再次 stop out。
2. 如果未来还要保留 re-entry 机制，默认方向应是“只保留更强确认的极少数回补”，而不是放松确认门槛。
3. 在没有新 clean 替代票之前，最安全的默认口径仍然是 `baseline` 当前的阻断策略。
