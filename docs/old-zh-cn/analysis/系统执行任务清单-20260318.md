# 2026-03-18 系统执行任务清单

## 1. 文档目的

这份文档把系统级路线图拆成可直接执行的 backlog。每项任务都只保留五个要素：目标、输入、输出、验收标准、建议顺序。

## 2. 执行原则

开始前先锁定四条原则：

1. 不做实盘部署准备。
2. 不做无新证据支撑的全局 Layer C、watchlist、avoid 放松实验。
3. 优先 fixed-artifact 分析、历史 reports 扫描和同口径 replay 对比。
4. 任何新结论都必须能落到现有索引、refresh protocol 和 validation 对比框架里。

## 8. backlog 完成的判断标准

这份 backlog 不要求系统立刻盈利，但要求做到三件事：

1. 每次结果都能稳定比较。
2. 每个主要问题都能落到清晰的责任层：候选、组合、退出、执行、数据。
3. 新样本进入系统时，有纪律地被分类、补证和吸收。
