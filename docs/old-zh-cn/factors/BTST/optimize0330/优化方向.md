如果只给一个结论：你现在最该做的，不是继续堆更多新因子，而是先把“漏掉的好票”稳定放出来，再把“入场质量”单独做成因子。当前最强证据在 btst_micro_window_regression_20260330.md 和 README.md。它们说明 baseline 在 0323-0326 闭环窗口里可执行样本为 0，但 false negative proxy 有 19 个；而 catalyst_floor_zero 已把 closed-cycle actionable 提升到 6，且 next_high_hit_rate、next_close_positive_rate、T+2 close positive rate 都到 0.8333。这个信号很明确：你的主问题不是市场没给机会，而是系统还在漏机会。

**先提胜率**
1. 固定当前 admission 基线，不要再回头做 shared Layer B 的大范围 floor 扫描。现阶段更该优化的是 short_trade_boundary score frontier 和结构性 blocked 样本，而不是重新找第二条 admission 主线。这个方向已经在 README.md 和 01-0330-research-execution-checklist.md 收口了。
2. 把执行 contract 收紧成双名单。selected 才进入可执行池，near_miss 只进观察池，不要再把观察票批量抬成主买点。你现在提高胜率的关键，不是“多放行”，而是“别把本该盯盘的票误当成交割票”。
3. 优先重做 breakout freshness 的使用方式，而不是继续扫阈值。最有效的做法是把它拆成两段：准备突破态先进入观察池，完成突破态才进入主执行池。这样能把很多本来会被一刀切掉、但次日盘中其实能确认的样本救回来。
4. 用 false negative 反推规则，不要只研究成功样本。你现在最有价值的三类样本就是：分数没过但次日 high 很强、只适合观察但盘中可交易、结构冲突重复出现但模式稳定。胜率提升，往往来自“少漏这种票”，不是来自“把 winner 特征越调越极端”。
5. 给所有新变体上硬门槛：新增样本必须大于 0，next_high_hit_rate@2% 不能低于 0.5217，next_close_positive_rate 不能低于 0.5652，而且不能被单一 ticker 或单一行业垄断。这样你才能防止“表面胜率变高，实则是在赌单票”。

**再提赔率**
1. 赔率不要先从延长持有天数下手，先从提高买点质量下手。当前最值得独立成因子的，是开盘消化能力、盘中确认质量、结构空间、板块共振密度、催化兑现节奏。这些在 README.md 已经列得很清楚。
2. 你真正该拉开的，不是“最后收没收红”，而是三段空间差：次日 high 相对前收的空间、次日 close 相对开盘的承接、次日 high 相对开盘的二次扩张。前两者决定你有没有安全垫，第三个决定你有没有赔率。
3. profitability 在 BTST 下更适合做条件化惩罚，而不是前置硬杀。当前文档已经说明，它对电子、电力设备、有色、通信等活跃行业压制过强，README.md 也明确建议改成软约束。你如果把它改成“高 breakout、高催化、强板块共振”条件下的软惩罚，通常会先改善赔率，再看能否同步改善胜率。
4. 样本治理也直接影响赔率。001309 这类可以当 primary controlled follow-through 模板，300383 只能当单票 shadow 模板，300724 继续冻结。三类样本不要混成一条 release 规则，否则左尾会被你自己放大，这一点在 01-0330-research-execution-checklist.md 已经定得很死。
5. 资金分配必须服从因子语义。主入场票可以集中，观察票默认不交易，shadow lane 只做验证不做放大。赔率不只是因子本身，更是“只把资金给最像主入场票的样本”。

**你现在的最优先顺序**
1. 先做 breakout freshness 的两段式改造，把准备突破态和完成突破态拆开。
2. 再做 profitability 的条件化软惩罚，只对高 breakout、高催化、强板块共振场景放松。
3. 最后才做新因子挖掘，而且只从 false negative archetype 里挖，不从少数 winner story 里挖。
