# Layer C 之前的短线候选，次日表现到底有没有变好：决策摘要

## 一句话结论

有，而且改善幅度不小。和旧的 `layer_b_boundary` 候选池相比，新的 `short_trade_boundary` 候选在当前真实样本里的次日表现明显更强；因此 Layer C 之前的下一轮主线，不应再优先怀疑“候选质量不行”，而应转向“如何在不显著损伤质量的前提下扩大覆盖”。

## 这次具体验证了什么

这次不是继续看静态因子，而是直接看 Layer C 之前候选的次日真实价格表现。

基于新脚本 `scripts/analyze_pre_layer_short_trade_outcomes.py`，对两组真实报告做了同口径比较：

1. 旧路径：`layer_b_boundary`
2. 新路径：`short_trade_boundary`

评估口径都是：T 日收盘形成候选后，看 T+1 的

- `next_open_return`
- `next_high_return`
- `next_close_return`

并额外统计：

- 次日最高价是否达到 `+2%`
- 次日收盘是否为正

## 旧候选池的表现

旧 `layer_b_boundary` 在当前 4 日窗口中共有 `23` 个候选拿到次日行情，表现如下：

- `next_high_return_mean = 0.0263`
- `next_close_return_mean = 0.0027`
- 次日最高价达到 `+2%` 的比例：`0.5217`
- 次日收盘为正的比例：`0.5652`

这说明旧池子不是“完全没用”，里面确实混着一些次日能动的票；但整体质量比较松散，均值层面只有很弱的次日收盘优势。

## 新候选池的表现

新 `short_trade_boundary` 在当前真实 builder 验证样本里共有 `2` 个候选拿到次日行情，表现如下：

- `next_high_return_mean = 0.0829`
- `next_close_return_mean = 0.0498`
- 次日最高价达到 `+2%` 的比例：`1.0`
- 次日收盘为正的比例：`1.0`

两个代表样本：

- `2026-03-24 / 300620`：次日最高 `+8.46%`，收盘 `+2.67%`
- `2026-03-24 / 001309`：次日最高 `+8.11%`，收盘 `+7.30%`

## 这意味着什么

这条证据把前置主线进一步收紧成下面两点：

1. 新 builder 已经成功把“进入 short-trade 补充池的候选质量”显著抬高了。
2. 当前 Layer C 之前的主问题，已经不再主要是“候选质量差”，而更像是“高质量候选数量太少”。

换句话说，如果现在继续把主要精力放在“再把 prefilter 变得更严”，大概率只会进一步压缩候选数，而不是提升真实次日表现。

## 但还要注意什么

这个结论是方向性很强，但样本量还不大：

- 旧池有 `23` 个样本
- 新池目前只有 `2` 个样本

所以这次结果更适合被解释为：

- “新 pre-Layer C builder 的方向是对的”
- 还不能被解释为“当前 builder 已经完成优化”

## 最合理的下一步

如果继续优先做 Layer C 之前的短线选股优化，下一轮最该做的不是继续打磨 Layer C blocked 释放，而是：

1. 围绕 `short_trade_boundary` 扩覆盖，而不是继续只做更严格过滤。
2. 重点审查哪些接近当前 floor 的候选被挡在补充池外，但次日其实可能仍有可交易表现。
3. 如果要调规则，优先做“增加高质量候选数”的小幅扩展，而不是进一步收紧 breakout / volume / catalyst floor。

## 最终决策

如果当前阶段先不看 Layer C，那么主线应切换为：继续优化 `short_trade_boundary` 这一层的候选覆盖率，并用次日表现去约束调参；不要再把主要精力放在 old `layer_b_boundary` 的质量补课上，也不要先回到 Layer C blocked 释放实验。
