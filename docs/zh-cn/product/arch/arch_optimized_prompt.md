# 架构优化策略

- 我建议我们目前的系统优化分两步走，把选股和如何买卖股票分到两步去迭代优化。这两大阶段，都有很多策略和优化的方法，混在一起互相影响，比如：股票选的不好，但是买卖点很好导致赚钱，我们会误以为股票选的也很好。如果最终效果不佳，很难分析根本原因是哪个因素造成的。所以，我们先从选股开始，把股票池选的尽量优秀，这样下一步买卖股票的基础就会很好，对优化股票买卖也会有很多帮助。
- 在第一步选股的时候，我希望按照系统已有的策略，每次筛选股票的结果，在提供给交易策略的同时，也导出一份markdown文档，列出最终筛选的股票和每只股票被选中的原因,然后我让专业的研究员去评审这些股票的质量，再反馈给你，看看有哪些方面需要调整。这样结合你的优化策略和外部人员的参考意见，可能会优化效果更好。

- 你觉得我上面的建议如何？是不是符合量化交易系统的优化方向？是不是符合业界最佳工业实践？我希望你能根据我的建议，现在我们这个系统的架构和业界最佳工业实践，帮我想一个不太复杂，又能非常有效的优化我的系统的方案。

# 写架构优化文档

- 很好，我希望你把上述内容，整理成一个架构设计文档。架构设计文档，要从底层原理，到方案为什么这么选择这样设计，以及如何设计都讲清楚，让人能够知道what，也能知道why和how，能够深入理解系统。你刚刚的解释已经包含了很多优秀的内容，可以在这基础上补充。写好架构设计文档，我们审阅几遍没有问题，我们再按照系统设计的最划算的版本，最低复杂度实施顺序，制定我们的实施计划。
