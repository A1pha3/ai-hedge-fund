# 次日短线目标指标与验证方案

## 0. 文档定位

本文档是 [arch_dual_target_selection_system.md](./arch_dual_target_selection_system.md) 的配套规格文档，专门回答一个问题：

短线交易目标到底如何衡量是否有效，以及如何在真实约束下验证它不是回测幻觉。

本文档不讨论完整系统架构，也不直接给出全部策略规则。本文档只聚焦：

1. 目标指标定义
2. 验证窗口与切片口径
3. 执行 realism 约束
4. 防过拟合机制
5. 默认档升级门槛

---

## 1. 设计原则

### 1.1 先定义指标，再谈调参

如果没有稳定指标，任何“感觉变好”都没有意义。

因此短线目标的研发顺序必须是：

1. 先定义 success metrics
2. 再定义验证窗口
3. 再允许调规则与阈值

### 1.2 指标必须服务真实交易，而不是服务纸面漂亮结果

短线目标的验证必须默认纳入：

1. T+1 制度
2. 涨跌停不可成交
3. 滑点与手续费
4. 流动性下限
5. 仓位与集中度约束

### 1.3 不以单一收益指标作为成败标准

单看收益会掩盖两个严重问题：

1. 风险可能不可接受
2. 策略可能完全不可扩容

因此必须同时看：

1. 收益
2. 回撤
3. 胜率
4. 盈亏比
5. 换手
6. 容量压力

---

## 2. 短线目标的正式目标函数

首版建议把短线目标函数定义为：

在 A 股 T+1 制度、真实执行约束与保守滑点假设下，最大化次日及短窗口交易的风险调整后正期望。

拆开后包含五个子目标：

1. 提升 T+1 正收益概率
2. 提升 T+1 盈亏比
3. 提升 T+1 可执行性
4. 降低极端回撤与连续亏损段
5. 保持不同市场状态下的稳定性

---

## 3. 指标分层

建议把指标分成四层：

1. 交易结果指标
2. 执行真实性指标
3. 稳定性指标
4. 研究治理指标

### 3.1 交易结果指标

核心指标：

1. T+1 open-to-close 胜率
2. T+1 open-to-high 中位数
3. T+1 open-to-close 中位数
4. T+3 持有窗口收益中位数
5. payoff ratio
6. expectancy

建议定义如下：

#### T+1 open-to-close 收益

定义：

$$
r_{oc}^{T+1} = \frac{Close_{T+1} - Entry_{T+1}}{Entry_{T+1}} - cost
$$

其中：

1. $Entry_{T+1}$ 默认为 T+1 开盘可成交价格或更保守的模拟成交价
2. $cost$ 包含佣金、印花税与滑点

#### T+1 open-to-high 收益

定义：

$$
r_{oh}^{T+1} = \frac{High_{T+1} - Entry_{T+1}}{Entry_{T+1}} - cost
$$

这个指标不用于表示可完全实现收益，而用于衡量次日弹性与赔率空间。

#### 胜率

定义：

$$
WinRate = \frac{\#(r > 0)}{N}
$$

#### 盈亏比

定义：

$$
PayoffRatio = \frac{|Median(r \mid r > 0)|}{|Median(r \mid r \le 0)|}
$$

#### 正期望

定义：

$$
Expectancy = WinRate \times AvgWin - (1 - WinRate) \times AvgLoss
$$

### 3.2 执行真实性指标

这些指标用于防止策略只在理想成交假设下成立。

必须记录：

1. 可成交率
2. 涨停无法买入率
3. 跌停无法退出压力率
4. 平均滑点损耗
5. 低流动性剔除占比

### 3.3 稳定性指标

这些指标用于判断策略是否只有局部窗口有效。

必须记录：

1. rolling 20-trading-day expectancy
2. rolling max drawdown
3. regime-by-regime WinRate
4. 参数微扰后的性能漂移
5. top decile 与 bottom decile 分层区分度

### 3.4 研究治理指标

这些指标用于判断策略是否可复盘、可审计。

建议记录：

1. short_trade selected count
2. hard-gate reject count
3. top blocker distribution
4. feedback 覆盖率
5. 规则版本命中分布

## 4. 评估窗口定义

### 4.1 主评估窗口

首版建议至少固定三个短线窗口：

1. T+1 open-to-close
2. T+1 open-to-high
3. T+3 持有窗口

原因：

1. open-to-close 代表保守可兑现结果
2. open-to-high 代表弹性与赔率上限
3. T+3 可检验“并非只是一日噪声”

### 4.2 辅助评估窗口

可选增加：

1. T+1 close-to-close
2. T+2 open-to-close
3. T+5 持有窗口

这些窗口主要用于识别策略是否偏向隔夜缺口、次日冲高回落，还是有更持久的短趋势。

---

## 5. 数据切片与 regime 验证

顶级系统不能只看全样本平均值，必须切片。

建议至少按以下维度切片：

1. 趋势市
2. 震荡市
3. 缩量市
4. 高波动市
5. 主题活跃市

同时按交易层面切片：

1. 大盘股
2. 中盘股
3. 小盘高波动股
4. 高流动性组
5. 低流动性边缘组

验证目标不是要求每个切片都极强，而是：

1. 不能在某关键 regime 下完全崩溃
2. 不能只靠单一板块或单一市况支撑全部结果

---

## 6. 执行 realism 假设

短线目标的回测必须默认使用保守假设。

建议首版硬性假设：

1. 仅使用 T 日已知信息生成 T+1 计划
2. T+1 开盘后才能视为实际入场
3. 涨停视为买入失败或高概率失败
4. 成交额不足样本直接过滤
5. 卖出时纳入跌停不可成交压力测试

### 6.1 入场价定义

建议使用两套口径并行：

1. base case：T+1 开盘成交价 + 基础滑点
2. stress case：T+1 开盘成交价 + 保守滑点惩罚

### 6.2 成本口径

建议至少纳入：

1. 买卖双边佣金
2. 卖出印花税
3. 基础滑点
4. 低流动性额外滑点

---

## 7. 验证框架

建议验证按四级推进：

### 7.1 Level 1：单日 frozen replay

目标：验证字段、gate 和 explainability 是否正确。

### 7.2 Level 2：短窗口 replay

目标：验证几天到几周窗口内 selected / rejected / blockers 的分布是否合理。

### 7.3 Level 3：walk-forward validation

目标：防止只在固定窗口上调参。

建议口径：

1. 用过去 N 天调参
2. 在后续 M 天验证
3. 滚动推进

### 7.4 Level 4：regime-sliced validation

目标：检验是否只在某一类市场状态下有效。

---

## 8. 防过拟合与参数治理

### 8.1 参数变更必须具备版本语义

每次规则调整必须记录：

1. 变更假设
2. 变更窗口
3. 改动前指标
4. 改动后指标
5. 是否通过样本外验证

### 8.2 阈值更新纪律

建议：

1. 单月仅允许有限次数默认阈值调整
2. 每次默认阈值升级都必须经过新窗口验证
3. 单窗口改善不得直接成为默认档依据

### 8.3 黑箱模型的前置条件

在以下条件满足前，不建议引入更复杂模型：

1. 指标定义稳定
2. 执行口径稳定
3. artifacts 链路稳定
4. 规则型基线已经建立

---

## 9. 默认档升级门槛

短线目标要从实验档升级到默认档，建议至少同时满足：

1. 多窗口 expectancy 为正
2. T+1 payoff ratio 稳定高于基线
3. 回撤未显著恶化
4. regime 切片中无明显崩溃区
5. 真实执行假设下仍保持边际优势
6. 人工复盘未发现大规模语义错误

如果只满足收益提高，但不满足回撤、稳定性或执行真实性，则不得升级为默认档。

---

## 10. artifacts 与报表要求

为了让指标体系可复用，建议每次短线目标实验输出：

1. 指标摘要 JSON
2. 窗口级 Markdown 报告
3. regime 切片明细表
4. selected / rejected / blockers ledger
5. 参数版本与 profile 信息

这会让后续评审重点从“口头解释”转移到“证据对照”。

---

## 11. 当前建议结论

短线目标的核心不是“找最猛的票”，而是：

1. 在真实约束下找正期望更高的次日交易机会
2. 用严格验证区分真实边际与回测幻觉
3. 用稳定指标和版本治理支撑持续迭代

如果没有这一套指标与验证框架，后续任何短线规则集都很容易再次退化成主观调参。
