# Pipeline 20 日窗口漏斗扫描分析

文档日期：2026 年 3 月 12 日  
分析范围：2026-02-02 至 2026-02-27 的 20 个交易日  
执行路径：默认 CLI / backtester pipeline 的真实筛选层  
说明：本次扫描使用已缓存的 candidate pool 快照，真实执行 Layer A、market state、score_batch、fuse_batch；未引入新的 workflow 结构试验。

## 1. 结论摘要

本轮 20 日窗口扫描给出了一条很明确的业务结论：

1. 默认 pipeline 的主要收缩点已经出现在 Layer B fast gate。
2. 20 个交易日里有 17 天 Layer B 通过数为 0。
3. 整个窗口只有 4 次 Layer B 入围，平均每天只有 0.2 只。
4. 没有出现 high_pool 被上限截断的情况，说明当前主问题不是池子过大，而是绝大多数股票根本过不了 `score_b >= 0.38`。

换句话说，当前交易稀少首先不是 Layer C 聚合、watchlist 过滤或下单约束造成的，而是前置 Layer B 门槛已经把绝大多数候选票挡在外面。

## 2. 窗口级汇总

窗口总览如下：

1. 交易日数：20
2. Layer B 为 0 的交易日：17
3. Layer B 大于 0 的交易日：3
4. Layer B 总入围数：4
5. Layer B 日均入围数：0.2
6. 聚合原因码：below_fast_score_threshold = 3996
7. 聚合原因码：high_pool_truncated_by_max_size = 0

这意味着 4000 个 Layer A 候选观察中，只有 4 个真正穿过了 0.38 的 fast gate。

## 3. 阈值拥挤分布

按整个 20 日窗口的 `score_b` 区间做粗聚合：

1. `[0.25, 0.30)`：142
2. `[0.30, 0.35)`：53
3. `[0.35, 0.38)`：9
4. `>= 0.38`：4

这组分布说明当前不是完全没有正向候选，而是存在明显的阈值拥挤：

1. 有一批票能稳定走到 0.25 以上。
2. 一部分票还能到 0.30 以上。
3. 但接近 0.38 时快速塌缩。
4. 真正穿过 fast gate 的样本极少。

因此，业务上更像是 Layer B 门槛过严或 Layer B 分数整体偏低，而不是后续层级把本来很多可交易标的再次过滤掉。

## 4. 按日摘要

有 Layer B 入围的只有 3 天：

1. 20260204：1 只入围
2. 20260205：2 只入围
3. 20260209：1 只入围

其余 17 天全部为 0。

几个代表性日期：

1. 20260202：Top1 `603993 = 0.3799`，距离阈值只差 0.0001，但最终仍为 0 只入围。
2. 20260203：Top1 `603993 = 0.3776`，Layer B 为 0。
3. 20260205：`300724 = 0.4717`、`000426 = 0.4268`，当天 Layer B 为 2。
4. 20260223：Top1 `300065 = 0.3735`，Layer B 为 0。
5. 20260224：Top1 `300065 = 0.3739`，Layer B 为 0。

从这些日期看，问题不是没有接近阈值的股票，而是大量股票停留在阈值下沿附近，没有足够多的票能稳定跨过 0.38。

## 5. 反复出现的入围 ticker

20 日窗口中真正穿过 Layer B 的 ticker 非常集中：

1. `000426`：2 次
2. `300724`：1 次
3. `600111`：1 次

这说明当前的 Layer B 输出不仅稀疏，而且集中度很高。业务上表现为：

1. 入围票覆盖面很窄。
2. 一旦少数 ticker 没有继续转单，组合就很容易退化到几乎无交易。

## 6. 当前最合理的业务判断

基于这次窗口扫描，当前最合理的业务判断是：

1. 不应优先怀疑 Layer C。
2. 不应优先怀疑下单器或日内执行约束。
3. 更不应重新回到调度结构实验。
4. 下一步应聚焦 Layer B gate 与 Layer B 分数形成机制。

但这里仍然要保持证据顺序：

1. 已确认主收缩点在 Layer B。
2. 尚未直接证明 0.38 阈值一定“过高”。
3. 还需要继续分解是四策略原始信号偏弱，还是融合/仲裁后整体被压低。

## 7. 建议的后续动作

基于当前证据，下一步优先级建议如下：

1. 对 Layer B 入围边缘样本做更细分解，按策略维度拆开 `trend / mean_reversion / fundamental / event_sentiment` 的贡献。
2. 对接近阈值但未入围的高频 ticker 做剖面分析，确认究竟是哪一类策略拖低了 `score_b`。
3. 在完成策略贡献拆解之前，不建议直接修改 `FAST_AGENT_SCORE_THRESHOLD`。
4. 如果后续证据仍指向阈值拥挤，再进入“小范围业务校准”，优先尝试围绕 Layer B gate 做最小改动，而不是改 workflow 结构。

## 8. Layer B 策略贡献拆解

对窗口内反复出现在 Top 区域的几个 ticker 做策略贡献拆解后，可以进一步收敛问题来源。

### 8.1 总体模式

边缘候选的 `score_b` 主要由两部分构成：

1. `fundamental`
2. `trend`

而另外两部分大多数时间贡献很弱：

1. `event_sentiment` 大部分 ticker 长期接近 0
2. `mean_reversion` 多数时候接近 0，少数日期为负，反而会轻微拉低总分

### 8.2 代表性 ticker 平均贡献

窗口内几个高频 ticker 的平均贡献大致如下：

1. `600111`：fundamental `0.2058`，trend `0.1193`，mean_reversion `-0.0036`
2. `300724`：fundamental `0.1874`，trend `0.1115`，mean_reversion `-0.0016`
3. `688008`：fundamental `0.1944`，trend `0.1133`，mean_reversion `-0.0026`
4. `603993`：fundamental `0.1362`，trend `0.0888`，mean_reversion `-0.0014`
5. `000426`：trend `0.1001`，fundamental `0.0394`，event_sentiment `0.0400`
6. `300065`：trend `0.0584`，fundamental `0.0441`，其余接近 0

这组结果说明：

1. Layer B 并不是四个策略都失效。
2. `fundamental + trend` 已经能持续提供正贡献。
3. 问题更像是总分抬升幅度不够，且缺乏第三个稳定的增量策略把边缘样本推过 0.38。

### 8.3 event_sentiment 的缺席

在这批高频候选里，`event_sentiment` 基本长期为 0，只有 `000426` 在个别日期出现明显正贡献。

这意味着：

1. 当前 Layer B 不太像“四策略协同推高分数”。
2. 更像是 `fundamental + trend` 两条腿在走路。
3. 一旦这两项中任意一项稍弱，整体就很容易停在 0.30 到 0.38 区间。

### 8.4 仲裁对边缘样本的影响

若看边缘 ticker 的逐日明细，还能看到仲裁规则会进一步压低部分样本：

1. `both_demote` 在 `300724`、`688008`、`600111` 的部分日期出现过
2. `trust_reversion` 曾把 `603993` 从接近阈值直接压回更低位置
3. `short_hold` / `long_hold` 只是持仓提示，不直接解释全部收缩，但它们和当前分数组合在一起时，并没有把边缘样本有效推过 fast gate

因此，后续如果继续拆 Layer B，除了看原始策略分数，还应把仲裁动作单独纳入分析，而不能只盯阈值本身。

### 8.5 0.35 到 0.38 边缘样本明细

对整个窗口里 `0.35 <= score_b < 0.38` 的样本单独抽取后，共得到 9 条边缘记录：

1. `603993`：20260202 `0.3799`，20260203 `0.3776`
2. `300724`：20260202 `0.3640`，20260206 `0.3614`
3. `300065`：20260223 `0.3735`，20260224 `0.3739`，20260225 `0.3736`
4. `000426`：20260206 `0.3609`
5. `688498`：20260205 `0.3516`

这些边缘样本的结构有两个值得注意的点：

1. `300065`、`688498` 这类样本根本没有触发会改写分数的仲裁动作，说明它们停在阈值下方主要是原始策略贡献不足，而不是仲裁把它们压下来。
2. `603993`、`300724`、`000426` 虽然带有 `short_hold` 或 `long_hold` 标记，但这两类动作在当前代码里只是持有周期提示，不直接修改 `score_b`，因此它们也不能解释为何样本没过 0.38。

换句话说，最接近阈值的这一批票，绝大多数不是被 score-mutating arbitration 打下来的，而是原始 Layer B 得分本来就没有足够越过 fast gate。

### 8.6 对后续优先级的影响

这条证据会直接影响后续排查顺序：

1. 如果只看边缘样本，优先级应先放在 Layer B 原始分数形成机制，而不是先改仲裁。
2. 仲裁仍然值得分析，但更像第二优先级，因为它解释的是一部分更低分样本，而不是 0.35 到 0.38 这批最接近过线的票。
3. 这进一步支持一个更谨慎的业务判断：在没有先拆透原始策略贡献之前，不应该直接把问题归咎于仲裁规则，也不应立即大幅下调 fast gate。

## 9. 子因子级别拆解

继续把 `0.35 <= score_b < 0.38` 的边缘样本下钻到 sub-factor 层后，问题进一步收敛。

### 9.1 event_sentiment 不是“弱”，而是大多数时间根本缺席

边缘样本的一个显著共同点是：

1. 除 `000426` 外，`event_sentiment` 的 completeness 基本都为 `0.0`
2. 这意味着不是这条策略给了弱正分或弱负分，而是它在多数边缘样本里没有形成有效输入
3. 只有 `000426` 在 20260206 通过 `insider_conviction` 提供了 `+0.05` 的增量，其余新闻相关子因子依然为 0

这说明当前 Layer B 更像是一个“缺少第三条腿”的结构：多数边缘样本只能依赖 trend 和 fundamental 两个大类，event_sentiment 没有稳定补分能力。

### 9.2 trend 内部：ema_alignment 很稳定，但 volatility 常常拖分

边缘样本中，trend 子因子呈现出相对稳定的模式：

1. `ema_alignment` 几乎总是稳定正贡献，常见值约 `+0.1125`
2. `adx_strength` 通常是小到中等正贡献
3. `momentum` 在部分股票上是强正贡献，但也有一些样本只有中性或接近 0
4. `volatility` 则经常是负项，例如 `603993` 约 `-0.075`，`300724` 约 `-0.0663`，`000426` 约 `-0.06`

这意味着 trend 本身不是简单的单边正向加分器。很多接近阈值的票，已经拿到了很强的均线排列和部分动量分，但同时被 volatility 明显往回拉。

### 9.3 fundamental 内部：profitability 和 financial_health 更稳定，growth 系列更分化

fundamental 子因子分布比表层更有信息量：

1. `profitability` 和 `financial_health` 在多数边缘样本中持续提供正贡献
2. `growth` 在不同股票之间分化明显，有的为正，有的为负，有的接近 0
3. `growth_valuation` 也不稳定，例如 `000426` 的该项为负
4. `industry_pe` 在这批边缘样本里基本长期为 0，没有形成有效区分度

代表性例子：

1. `603993`：profitability、financial_health、growth_valuation 为正，但 `growth` 为负，抵消了一部分 fundamental 总分
2. `300724`：fundamental 结构相对完整，profitability、growth、financial_health、growth_valuation 都是正项
3. `300065`：financial_health、growth 为正，但 profitability 明显为负，导致总分始终卡在阈值下方
4. `000426`：profitability 为正，但 growth 与 financial_health 基本不给增量，growth_valuation 还是负项

### 9.4 mean_reversion 基本不是主驱动项

边缘样本里，mean_reversion 大多数时候都不构成主要决定因素：

1. 很多票的 `zscore_bbands`、`rsi_extreme`、`stat_arb`、`hurst_regime` 都接近 0
2. 即便像 `300065` 这样有一点 `stat_arb` 正贡献，也会被 `hurst_regime` 的负贡献大致抵消

因此，当前窗口里的边缘样本并没有体现出“均值回归子因子明显被压制，导致整体不过线”的特征。

### 9.5 目前最有价值的业务解释

把子因子层证据和前面的策略层证据合起来，当前最有价值的解释是：

1. Layer B 的边缘样本通常已经具备较强的 trend / fundamental 主干信号
2. 但 trend 会被 volatility 回拉，fundamental 会被 growth 或 growth_valuation 的分化项削弱
3. event_sentiment 又大多直接缺席，无法提供第三个稳定增量
4. 最终使得大量样本停在 `0.35 ~ 0.38` 下沿，而不是顺利越过 fast gate

## 10. 当前业务主线的下一步

到这一步，下一轮分析的优先级已经可以再缩小：

1. 优先检查为什么边缘样本的 `event_sentiment` completeness 长期为 0
2. 同时检查 trend 的 `volatility` 负贡献是否系统性偏强
3. 再检查 fundamental 的 `growth` / `growth_valuation` 是否在某些类别股票上过于保守

这三个问题都比“直接下调 fast gate”更值得先回答。

## 11. 本次分析的定位

这份扫描的价值，不是直接给出最终调参结论，而是把业务主线进一步收敛到一个更小的问题集合：

1. 不是 provider 速度问题。
2. 不是 rolling scheduler 问题。
3. 不是 Layer C 主导的问题。
4. 当前最值得深挖的是 Layer B 分数分布、仲裁动作与 fast gate 的相互作用。

## 12. event_sentiment 缺席的进一步定位

针对前面识别出的边缘样本，又继续下钻到了 A 股新闻源本身，结论比前一轮更明确。

### 12.1 不是 event stage truncation

前面的运行时检查已经证明：

1. `603993`、`300724`、`300065`、`000426`、`688498` 这些边缘样本都确实进入了 `fundamental_candidates`
2. 它们也都进入了 `event_candidates`
3. 因此 `event_sentiment` completeness 为 0 的直接原因，不是 Layer B 内部的候选截断

### 12.2 也不是本地 relevance filter 把历史新闻清空

继续直接检查 `ak.stock_news_em(symbol=ticker)` 的原始返回后，发现对这些样本股票：

1. 接口返回条数非常小，通常只有 10 条左右
2. 返回内容主要是当前可见的最近新闻，而不是可按 `start_date/end_date` 查询的历史新闻档案
3. 在 2026-02 的回测窗口里，这批返回结果大多完全落在窗口之外，因此进入 `get_ashare_company_news()` 后会先被日期过滤掉
4. 对原始标题逐条检查后，可以看到多数记录即使经过 `_is_news_relevant_to_stock()` 也会被判为相关，因此主因不是 relevance filter

换句话说，当前 A 股新闻链路在历史回测语境下的主要问题不是“过滤过严”，而是“数据源本身没有提供对应窗口内的历史新闻”。

### 12.3 运行时证据

这轮检查里有两个关键证据：

1. 对代表性边缘 ticker，`ak.stock_news_em()` 的原始结果总量很小，而且发布时间集中在 2026-03 的最近几天
2. 把这些原始结果代入 2026-02 的 30 日 event window 后，`window_rows` 基本全部为 0

例如：

1. `603993` 原始新闻虽然存在，但返回的 10 条几乎都在 2026-03，落不到 2026-02-02 的回测窗口内
2. `300724`、`300065`、`000426` 也表现出同样模式
3. `688498` 仅出现过一条 2026-02-10 的较早新闻，因此在 `20260205` 的窗口里依然拿不到任何新闻输入

这就解释了为什么前面看到的大多数边缘样本是：

1. `news_count = 0`
2. `trade_count = 0`
3. `event_sentiment` 的 `news_sentiment` 和 `event_freshness` completeness 同时为 0

### 12.4 对业务判断的影响

这个结论会改变后续排查和调参顺序：

1. 当前不能把 A 股回测里的 `event_sentiment` 缺席简单解释成策略弱或关键词规则差
2. 至少在当前数据源形态下，它更像是一个“历史新闻不可回放”的数据可得性问题
3. 因而如果直接基于当前回测结果去调 `event_sentiment` 权重、关键词或 fast gate，很容易把数据源缺口误判成策略缺陷

### 12.5 当前最稳妥的下一步

基于现有证据，更稳妥的后续方向是：

1. 先把 A 股 `event_sentiment` 视为“历史新闻覆盖受限”的已知约束，而不是立刻调策略参数
2. 单独评估 insider-trade 数据能否成为更稳定的 A 股 event 输入
3. 若后续要继续提升 A 股历史回测可信度，应优先补“可按日期回放的新闻数据源或快照”，再讨论 event 相关策略校准

## 13. volatility 子因子的反事实测算

在确认 `event_sentiment` 的主要问题是历史新闻不可回放之后，又继续对 `trend.volatility` 做了一个更直接的反事实测算：

1. 使用同一组 2026-02 的 20 个交易日 candidate pool 快照
2. 重放真实 `score_batch -> fuse_signals_for_ticker` 链路
3. 仅在反事实分支里，把 `trend` 中 direction 为负的 `volatility` 子因子从聚合中移除
4. 不改 market state、不改仲裁、不改 fast gate，只看 `score_b` 和通过数的变化

### 13.1 总体结果

这个测算给出一个很关键的业务结论：

1. baseline 的 Layer B 总通过数仍然是 4
2. 去掉 negative volatility 后，20 日窗口总通过数上升到 11
3. 净增通过数为 7
4. 新增穿线样本分布在 5 个交易日：20260202、20260203、20260206、20260225、20260227

也就是说，negative volatility 不是一个边缘噪音项，而是足以把窗口总通过数从 4 推到 11 的实质性拖分因子。

### 13.2 它很常见，但不是所有边缘样本都会受影响

20 日窗口内，在进入评分链路的 top-160 技术阶段样本里：

1. 共出现了 1076 个带 negative volatility 的样本
2. 平均每个交易日约有 54 个左右
3. 但真正从“阈值下方穿到阈值上方”的只有 7 个

这说明：

1. negative volatility 在当前市场环境里并不少见
2. 但它不是“所有交易稀少问题的唯一根因”
3. 更准确的说法是：它会系统性压低一批本来已经接近 fast gate 的票，但对那些原始分数更低的样本，去掉这一项也不够

### 13.3 对边缘样本的直接影响

最有代表性的边缘样本变化如下：

1. `603993`：20260202 从 `0.3799` 升到 `0.4486`，20260203 从 `0.3776` 升到 `0.4447`
2. `300724`：20260202 从 `0.3640` 升到 `0.4001`，20260206 从 `0.3614` 升到 `0.3974`
3. `000426`：20260206 从 `0.3609` 升到 `0.3963`

这些样本有一个共同点：

1. 原始 trend 方向本来就是正的
2. 但 `volatility` 子因子给出明显负向信号
3. 去掉这一下拉项后，它们就能穿过 `0.38` fast gate

从指标上看，这类样本常伴随：

1. `volatility_regime > 1.5`
2. `volatility_z_score > 2.5`
3. 某些样本的 `vol_confidence` 直接达到或接近 100

也就是说，当前 volatility 规则对“高波动但仍在上行趋势中的股票”打分相当重。

### 13.4 也有一批边缘样本几乎不受 volatility 影响

反事实结果同样给出一个反证：并不是所有边缘票都能靠去掉 volatility 过线。

代表性例子：

1. `300065`：20260223 到 20260225 连续三天都停在 `0.3735 ~ 0.3739`，去掉 volatility 后几乎没有变化
2. `688498`：20260205 的 `0.3516` 在反事实里也没有抬升

原因是这些样本的 `volatility` 子因子本来就是中性，而不是负项。

这说明先前文档里的另一个判断仍然成立：

1. 对一部分边缘票，主问题确实是 volatility 压分
2. 但对另一部分边缘票，根因仍然在 fundamental 的 growth / growth_valuation 分化，或者 event 输入缺席

### 13.5 对当前优先级的影响

到这一步，业务优先级可以进一步明确为：

1. `event_sentiment`：数据源历史覆盖问题，优先按数据约束理解，不宜先调参数
2. `volatility`：已经被反事实验证为真实且显著的次主因，值得优先进入规则校准讨论
3. `growth / growth_valuation`：仍是那些“不受 volatility 影响却依然卡在阈值下方”的样本的下一层重点

因此，下一轮如果继续深挖 Layer B，最合理的动作不是直接整体下调 fast gate，而是：

1. 先检查 volatility 的 bearish 触发条件和置信度斜率是否对趋势股过于激进
2. 再把剩余未解释的边缘样本转交给 fundamental 子因子继续拆解

## 14. 剩余边缘样本的 fundamental 分型

在把 `event_sentiment` 和 `negative volatility` 两条线都拆开之后，剩下还卡在阈值下方的样本不再是同一种问题。

这轮继续对最典型的两类样本做了 fundamental 子因子反事实：

1. `300065`：连续三天都在 `0.3735 ~ 0.3739`
2. `688498`：20260205 停在 `0.3516`

### 14.1 300065：主问题不是 growth，而是 profitability 硬负项

对 `300065` 来说，fundamental 内部结构非常明确：

1. `growth` 为正，score 约 `0.65`
2. `financial_health` 为强正，score 为 `1.0`
3. `growth_valuation` 只是中性，不是负项
4. 真正拉低 fundamental 的是 `profitability`

它的 `profitability` 子因子连续三天都是：

1. direction = `-1`
2. confidence = `100`
3. 触发原因很直接：`ROE`、`net_margin`、`operating_margin` 三项全部低于当前阈值

对应数值大致是：

1. `return_on_equity ≈ 0.0216`
2. `net_margin ≈ 0.0533`
3. `operating_margin ≈ 0.0541`

做反事实后，如果只移除这个负向 `profitability` 子因子：

1. 20260223：`0.3735 -> 0.3870`
2. 20260224：`0.3739 -> 0.3874`
3. 20260225：`0.3736 -> 0.3871`

也就是说，`300065` 不是被 growth 或估值项拖住，而是被一个相当刚性的“低盈利质量”判断卡在阈值下方。

### 14.2 688498：不是单个 fundamental 负项，而是缺少第三条增量腿

`688498` 的结构正好相反。

它在 20260205 的 fundamental 并不差：

1. `profitability` 为正
2. `growth` 为正
3. `financial_health` 为强正
4. `growth_valuation` 只是中性
5. 没有任何 fundamental 子因子是显式负项

但即便如此，它的总分仍停在 `0.3516`。原因是：

1. `trend` 贡献约 `0.2054`
2. `fundamental` 贡献约 `0.1461`
3. `mean_reversion` 为中性，贡献接近 `0`
4. `event_sentiment` 缺席，贡献为 `0`

所以 `688498` 的问题不是“有哪个 fundamental 子因子明显打了负分”，而是：

1. 两条主腿 `trend + fundamental` 虽然都为正
2. 但没有第三个策略继续把分数往上推
3. 最终就停在 `0.35` 左右，仍然过不了 `0.38`

### 14.3 这对后续校准顺序的意义

这一步很重要，因为它说明剩余边缘票至少已经分成两种类型：

1. `300065` 这一类：主要受某个明确的负向 fundamental 子因子压制，本例中是 `profitability`
2. `688498` 这一类：没有明显 fundamental 负项，但由于 `event_sentiment` 缺席、`mean_reversion` 中性，缺少第三个增量策略，导致总分自然停在阈值下方

因此，如果后续继续做最小化业务校准，顺序应更谨慎：

1. 不应把所有边缘票统一归因到 growth 或 growth_valuation
2. 对 `300065` 一类，应重点审视 `profitability` 的阈值是否对某些高成长但低净利率行业过于刚性
3. 对 `688498` 一类，则更应该优先回到“event 输入缺席”和“第三策略缺口”这个问题，而不是单独放松 fundamental 规则

### 14.4 000426：轻度估值惩罚型，而不是 profitability 问题

再补一个对照样本后，边缘分型可以进一步细化。

`000426` 在 20260206 的结构与 `300065`、`688498` 都不同：

1. `profitability` 是强正项，不是问题来源
2. `growth` 只是中性，既没有明显加分，也没有显著拖分
3. `financial_health` 也是中性
4. 真正的负项来自 `growth_valuation`

它当日的关键估值指标是：

1. `peg_ratio ≈ 10.82`
2. `price_to_sales_ratio ≈ 16.81`
3. `growth_valuation` direction = `-1`，confidence = `65`

做反事实后，如果只移除这个负向 `growth_valuation` 子因子：

1. `score_b` 只会从 `0.3609` 抬到 `0.3673`
2. 增量约 `+0.0064`
3. 仍然不足以穿过 `0.38`

这说明 `000426` 并不是被单一估值负项“严重压死”的样本，而更像是：

1. 有一小块明确的估值惩罚
2. 同时 `growth` 和 `financial_health` 又没有形成足够强的正向补分
3. 叠加 event 侧虽然有一点 insider 信号，但仍不足以把它推过阈值

因此，边缘样本实际上已经可以分成至少三类：

1. 硬负项压线型：如 `300065`，核心是 `profitability`
2. 轻度估值惩罚型：如 `000426`，有负项但单独去掉也不够穿线
3. 第三条腿缺失型：如 `688498`，没有明显 fundamental 负项，但整体缺少额外增量策略

## 15. profitability 规则本身的 cliff 特征

继续往下看 `profitability` 后，可以把问题再说得更精确一些：当前它不是一个平滑惩罚项，而是一个非常明显的二元阈值 cliff。

### 15.1 当前规则的结构

`profitability` 只看三项：

1. `ROE >= 0.15`
2. `net_margin >= 0.20`
3. `operating_margin >= 0.15`

然后直接做离散判定：

1. 至少 2 项达标：direction = `+1`
2. 0 项达标：direction = `-1`
3. 只有 1 项达标：direction = `0`

这意味着它不是连续扣分，而是一个“跨不过门槛就直接翻负”的结构。

### 15.2 正反样本对照

几个代表性样本把这个 cliff 特征展示得很清楚：

1. `300724`，行业是 `电力设备`，在 20260205 已经通过 Layer B，`ROE ≈ 0.2566`、`net_margin ≈ 0.1745`、`operating_margin ≈ 0.2026`
2. 它虽然 `net_margin` 没到 0.20，但因为 `ROE` 和 `operating_margin` 达标，所以 `profitability` 直接为正
3. `300065`，行业是 `国防军工`，在 20260223 的三项分别约为 `0.0216 / 0.0533 / 0.0541`，3 项全部未达标，因此 `profitability` 直接成为 direction = `-1`、confidence = `100` 的硬负项
4. `600111`，行业是 `有色金属`，在 20260204 的三项约为 `0.0896 / 0.0513 / 0.0822`，同样 3 项全部未达标，因此也被同一规则直接判成强负项

这说明问题并不是只集中在某一个行业，而是：

1. 只要股票处在“成长性不错，但盈利率尚未跨过这三条线”的状态
2. 当前规则就会给出相当刚性的负面结论

### 15.3 反事实结果进一步说明它不是小噪音

对 `600111` 做反事实时，如果只移除 `profitability` 负项：

1. `score_b` 会从 `0.3454` 抬到 `0.3803`
2. 直接越过 fast gate

而它当时的其它 fundamental 子因子其实并不弱：

1. `growth` 为正
2. `financial_health` 为正
3. `growth_valuation` 也为正

这意味着 `profitability` 在这类样本里不是“锦上添花的微调项”，而是真正决定能否过线的关键闸门。

### 15.4 对当前业务判断的影响

这一层证据会把前面的判断再向前推进一步：

1. `profitability` 的问题，不应再简单理解成个别股票基本面差
2. 更准确的说法是：当前规则对“低利润率但其它因子较强”的股票存在明显 cliff 效应
3. 这种效应既会出现在 `国防军工` 这类成长行业，也会出现在 `有色金属` 这类周期行业

因此，如果后续要继续做最小化业务校准，`profitability` 应该被视为一个独立优先项：

1. 优先检查 3 个阈值是否过于刚性
2. 优先检查 direction 从 `0 项达标 -> 直接强负` 是否太陡
3. 在这一步做清楚之前，不宜把大量边缘票继续笼统归因给 fast gate 本身

### 15.5 小范围反事实校准：移除硬负项有效，弱化硬负项反而无效

基于前面已经确认的 4 个 `profitability` 硬负项样本，又做了一步更贴近“实际调参”的小范围反事实：

样本包括：

1. `300065`：20260223、20260224、20260225
2. `600111`：20260204

比较了两种处理方式：

1. `remove_hard_negative`：把 `profitability` 这个硬负项直接从 fundamental 聚合中移除
2. `weaken_hard_negative`：保留负方向，但把置信度从 `100` 降到 `40`

结果非常有价值，也有一点反直觉：

1. `remove_hard_negative` 是有效的
2. `weaken_hard_negative` 在当前聚合规则下反而会让 `score_b` 更低

具体数值：

1. 对 `300065` 三个样本，移除硬负项后都从约 `0.3735 ~ 0.3739` 升到约 `0.3870 ~ 0.3874`
2. 对 `600111`，移除硬负项后从 `0.3454` 升到 `0.3803`
3. 但如果只是把负项置信度降到 `40`，`300065` 反而会掉到约 `0.3404 ~ 0.3408`
4. `600111` 也会从 `0.3454` 进一步掉到 `0.2958`

这说明一个关键事实：

1. 在当前 `aggregate_sub_factors()` 规则下，简单“减弱负项置信度”并不等价于“减少负项伤害”
2. 因为该聚合器同时用方向加权和整体置信度/一致性来决定最终输出
3. 某些情况下，把一个明确的强负项改成低置信度负项，会让最终子策略的正方向置信度更差，而不是更好

因此，如果后续真的要对 `profitability` 做最小化业务校准，候选方向更像是：

1. 调整硬负项触发条件，让更少股票落入 `0 项达标 -> 强负` 这条支路
2. 或者改变这类硬负项在子因子聚合里的参与方式
3. 而不是简单把 `confidence=100` 改成更低数值

## 16. 第三条腿缺失型里，mean_reversion 的“中性稀释”效应

继续往下看 `688498` 这类“第三条腿缺失型”后，又出现了一个非常重要的机制层发现：

1. `mean_reversion` 的问题不只是“没有提供正贡献”
2. 更关键的是，它以 `completeness = 1.0` 的中性策略身份进入了 Layer B 融合
3. 这会在权重归一化时，实际稀释已经为正的 `trend` 和 `fundamental`

### 16.1 688498 的 mean_reversion 结构

在 20260205，`688498` 的 `mean_reversion` 几乎是一个“全中性包”：

1. `zscore_bbands` 中性
2. `rsi_extreme` 中性
3. `stat_arb` 中性
4. `hurst_regime` 中性

但它们都带着：

1. `completeness = 1.0`
2. 置信度大约在 `42 ~ 50`

结果是：

1. `mean_reversion` 最终 direction = `0`
2. confidence 仍然约为 `48.45`
3. completeness 仍然是 `1.0`

### 16.2 这不是“零影响”，而是“稀释正项”

当前权重归一化逻辑只要看到 completeness 大于 0，就会把该策略纳入可用权重集合。

因此对 `688498` 来说：

1. `trend` 为正
2. `fundamental` 为正
3. `mean_reversion` 虽然方向是 0，但仍参与归一化
4. 结果 `trend` 和 `fundamental` 的有效权重都被摊薄

做一个简单反事实，把这些中性 `mean_reversion` 子因子从聚合中移除后：

1. `688498` 的 `score_b` 会从 `0.3516` 直接升到 `0.4688`
2. 增量达到 `+0.1172`
3. 这已经远大于前面 `000426` 的轻度估值惩罚增量，也大于很多单一 fundamental 子因子的影响

换句话说，`688498` 表面上看像是“第三条腿缺失”，本质上其实更接近：

1. 第三条腿不仅缺失
2. 还以一个完整中性策略的形式稀释了本来已经有效的两条正腿

### 16.3 300065 的对照

这个机制并不只存在于 `688498`。

对 `300065` 而言，`mean_reversion` 不是完全中性，而是一个很弱的正项：

1. `stat_arb` 为小幅正向
2. `hurst_regime` 为负向
3. 综合后只剩 direction = `+1`、confidence ≈ `10.2`

如果把其中的中性子因子移出聚合，`300065` 的 `score_b` 也会从 `0.3735` 升到 `0.3849`

这说明：

1. `mean_reversion` 当前不只是“偶尔不给 alpha”
2. 它在不少样本里还可能通过中性 completeness 参与权重分配，形成结构性稀释

### 16.4 对后续业务校准的影响

这条证据会明显改变后续优先级：

1. 对 `688498` 这类样本，问题不应再简单概括成“event_sentiment 缺席”
2. 更准确的说法是：`event_sentiment` 缺席叠加 `mean_reversion` 中性稀释，共同导致第三条腿失效
3. 如果后续进入规则校准，`mean_reversion` 的 completeness 处理方式应被列为和 `profitability` cliff 同一优先级的问题

因此，当前最值得继续验证的不是单纯“提高 mean_reversion 正向得分”，而是：

1. 中性策略是否应该在 Layer B 权重归一化时占用完整权重
2. 哪些子因子组合下，应该把 `direction = 0` 但 `completeness = 1` 的策略降级为“信息不足”而不是“完整中性”

### 16.5 窗口级保守反事实：只排除整条中性 mean_reversion 策略，影响仍然很大

为了避免前面的单样本结论过度夸大，又做了一次更保守的窗口级反事实：

1. 不再逐个删除所有中性子因子
2. 只在 `mean_reversion` 整体结果满足 `direction = 0 且 completeness > 0` 时
3. 才把整条 `mean_reversion` 策略从 Layer B 权重归一化里排除

即便用这个更克制的版本，20 日窗口结果仍然非常大：

1. baseline 的 Layer B 通过数仍然只有 `4`
2. 反事实后通过数上升到 `69`
3. 新增穿线样本达到 `65`
4. 共影响了 `2026` 个样本的权重归一化

这说明一个比单样本更稳的事实：

1. `mean_reversion` 的中性 completeness 不是偶发现象
2. 它在当前 20 日窗口里是非常普遍的结构状态
3. 一旦把这类“完整中性策略”从权重归一化中移除，Layer B 通过率会被大幅改写

### 16.6 这不是直接的调参建议，而是边界测试

这个结果的量级非常大，因此不能直接等同于“应该按这个方式上线改规则”。

更合理的解释是：

1. 当前 Layer B 对 `mean_reversion` 的中性 completeness 非常敏感
2. 这暴露出权重归一化层面存在明显的结构性放大效应
3. 但“整条中性策略全部排除”本身仍然是一个偏激的反事实边界测试

因此，这轮分析给出的业务含义不是“直接删除中性 mean_reversion”，而是：

1. 后续如果要做最小化规则改动，应优先研究更温和的 completeness 语义
2. 例如中性策略是否只占部分权重、或仅在部分子因子真正有有效信息时才参与归一化
3. 在没有先做这种温和版本验证之前，不应把本轮 `4 -> 69` 的反事实结果直接当成可上线的目标值

### 16.7 更温和的 completeness cap 在当前实现里其实无效

为了验证能否用一个更保守的办法缓解这个问题，又做了两组更温和的反事实：

1. 当 `mean_reversion` 整体为中性时，把它的 completeness 从 `1.0` 降到 `0.5`
2. 当 `mean_reversion` 整体为中性时，把它的 completeness 从 `1.0` 降到 `0.25`

结果两组都完全没有变化：

1. 20 日窗口 Layer B 通过数仍然是 `4`
2. 新增穿线数仍然是 `0`
3. 连边缘样本如 `603993`、`688498`、`000426`、`300724` 都没有任何抬升

这条证据非常关键，因为它把问题从“业务参数”进一步收敛到了“实现语义”：

1. 当前归一化逻辑只检查 `completeness > 0`
2. 只要大于 0，这条策略就会被视为 fully active，参与完整的权重归一化
3. 因此把 completeness 从 `1.0` 改到 `0.5` 或 `0.25`，在当前实现里并不会减少它对 Layer B 的权重占用

换句话说，当前代码里的 completeness 在 Layer B 融合层面并不是一个连续权重，而更像一个二值开关。

### 16.8 这对最小化规则改动草案的意义

到这里，最小化改动方向已经进一步清晰：

1. 如果要缓解 `mean_reversion` 的中性稀释问题，光改 completeness 数值本身不够
2. 必须改 `_normalize_for_available_signals()` 对 active signal 的判定方式，或者改 `direction = 0` 策略进入归一化的资格
3. 这与前面 `profitability` 的结论一致：问题不只在参数值，也在当前聚合语义本身

因此，从分析角度看，当前最小化规则修改草案可以收敛到两条：

1. `profitability`：不要只降 confidence，应优先调整“0 项达标 -> 强负”的进入条件或其聚合参与方式
2. `mean_reversion`：不要只降 completeness 数值，应优先调整“中性但 completeness>0 的策略是否占用完整归一化权重”
