# Pipeline 20 日窗口漏斗扫描分析

文档日期：2026 年 3 月 12 日  
分析范围：2026-02-02 至 2026-02-27 的 20 个交易日  
执行路径：默认 CLI / backtester pipeline 的真实筛选层  
说明：本次扫描使用已缓存的 candidate pool 快照，真实执行 Layer A、market state、score_batch、fuse_batch；未引入新的 workflow 结构试验。

## 1. 结论摘要

本轮 20 日窗口扫描给出了一条很明确的业务结论：

1. 默认 pipeline 的主要收缩点已经出现在 Layer B fast gate。
2. 20 个交易日里有 17 天 Layer B 通过数为 0。
3. 整个窗口只有 4 次 Layer B 入围，平均每天只有 0.2 只。
4. 没有出现 high_pool 被上限截断的情况，说明当前主问题不是池子过大，而是绝大多数股票根本过不了 `score_b >= 0.38`。

换句话说，当前交易稀少首先不是 Layer C 聚合、watchlist 过滤或下单约束造成的，而是前置 Layer B 门槛已经把绝大多数候选票挡在外面。

## 2. 窗口级汇总

窗口总览如下：

1. 交易日数：20
2. Layer B 为 0 的交易日：17
3. Layer B 大于 0 的交易日：3
4. Layer B 总入围数：4
5. Layer B 日均入围数：0.2
6. 聚合原因码：below_fast_score_threshold = 3996
7. 聚合原因码：high_pool_truncated_by_max_size = 0

这意味着 4000 个 Layer A 候选观察中，只有 4 个真正穿过了 0.38 的 fast gate。

## 3. 阈值拥挤分布

按整个 20 日窗口的 `score_b` 区间做粗聚合：

1. `[0.25, 0.30)`：142
2. `[0.30, 0.35)`：53
3. `[0.35, 0.38)`：9
4. `>= 0.38`：4

这组分布说明当前不是完全没有正向候选，而是存在明显的阈值拥挤：

1. 有一批票能稳定走到 0.25 以上。
2. 一部分票还能到 0.30 以上。
3. 但接近 0.38 时快速塌缩。
4. 真正穿过 fast gate 的样本极少。

因此，业务上更像是 Layer B 门槛过严或 Layer B 分数整体偏低，而不是后续层级把本来很多可交易标的再次过滤掉。

其余 17 天全部为 0。

从这些日期看，问题不是没有接近阈值的股票，而是大量股票停留在阈值下沿附近，没有足够多的票能稳定跨过 0.38。

## 5. 反复出现的入围 ticker

20 日窗口中真正穿过 Layer B 的 ticker 非常集中：

1. `000426`：2 次
2. `300724`：1 次
3. `600111`：1 次

这说明当前的 Layer B 输出不仅稀疏，而且集中度很高。业务上表现为：

1. 入围票覆盖面很窄。
2. 一旦少数 ticker 没有继续转单，组合就很容易退化到几乎无交易。

## 6. 当前最合理的业务判断

基于这次窗口扫描，当前最合理的业务判断是：

1. 不应优先怀疑 Layer C。
2. 不应优先怀疑下单器或日内执行约束。
3. 更不应重新回到调度结构实验。
4. 下一步应聚焦 Layer B gate 与 Layer B 分数形成机制。

但这里仍然要保持证据顺序：

1. 已确认主收缩点在 Layer B。
2. 尚未直接证明 0.38 阈值一定“过高”。
3. 还需要继续分解是四策略原始信号偏弱，还是融合/仲裁后整体被压低。

## 7. 建议的后续动作

基于当前证据，下一步优先级建议如下：

1. 对 Layer B 入围边缘样本做更细分解，按策略维度拆开 `trend / mean_reversion / fundamental / event_sentiment` 的贡献。
2. 对接近阈值但未入围的高频 ticker 做剖面分析，确认究竟是哪一类策略拖低了 `score_b`。
3. 在完成策略贡献拆解之前，不建议直接修改 `FAST_AGENT_SCORE_THRESHOLD`。
4. 如果后续证据仍指向阈值拥挤，再进入“小范围业务校准”，优先尝试围绕 Layer B gate 做最小改动，而不是改 workflow 结构。

## 8. Layer B 策略贡献拆解

对窗口内反复出现在 Top 区域的几个 ticker 做策略贡献拆解后，可以进一步收敛问题来源。

### 8.1 总体模式

边缘候选的 `score_b` 主要由两部分构成：

1. `fundamental`
2. `trend`

而另外两部分大多数时间贡献很弱：

1. `event_sentiment` 大部分 ticker 长期接近 0
2. `mean_reversion` 多数时候接近 0，少数日期为负，反而会轻微拉低总分

### 8.2 代表性 ticker 平均贡献

窗口内几个高频 ticker 的平均贡献大致如下：

1. `600111`：fundamental `0.2058`，trend `0.1193`，mean_reversion `-0.0036`
2. `300724`：fundamental `0.1874`，trend `0.1115`，mean_reversion `-0.0016`
3. `688008`：fundamental `0.1944`，trend `0.1133`，mean_reversion `-0.0026`
4. `603993`：fundamental `0.1362`，trend `0.0888`，mean_reversion `-0.0014`
5. `000426`：trend `0.1001`，fundamental `0.0394`，event_sentiment `0.0400`
6. `300065`：trend `0.0584`，fundamental `0.0441`，其余接近 0

这组结果说明：

1. Layer B 并不是四个策略都失效。
2. `fundamental + trend` 已经能持续提供正贡献。
3. 问题更像是总分抬升幅度不够，且缺乏第三个稳定的增量策略把边缘样本推过 0.38。

### 8.3 event_sentiment 的缺席

在这批高频候选里，`event_sentiment` 基本长期为 0，只有 `000426` 在个别日期出现明显正贡献。

这意味着：

1. 当前 Layer B 不太像“四策略协同推高分数”。
2. 更像是 `fundamental + trend` 两条腿在走路。
3. 一旦这两项中任意一项稍弱，整体就很容易停在 0.30 到 0.38 区间。

### 8.4 仲裁对边缘样本的影响

若看边缘 ticker 的逐日明细，还能看到仲裁规则会进一步压低部分样本：

1. `both_demote` 在 `300724`、`688008`、`600111` 的部分日期出现过
2. `trust_reversion` 曾把 `603993` 从接近阈值直接压回更低位置
3. `short_hold` / `long_hold` 只是持仓提示，不直接解释全部收缩，但它们和当前分数组合在一起时，并没有把边缘样本有效推过 fast gate

因此，后续如果继续拆 Layer B，除了看原始策略分数，还应把仲裁动作单独纳入分析，而不能只盯阈值本身。

### 8.5 0.35 到 0.38 边缘样本明细

对整个窗口里 `0.35 <= score_b < 0.38` 的样本单独抽取后，共得到 9 条边缘记录：

1. `603993`：20260202 `0.3799`，20260203 `0.3776`
2. `300724`：20260202 `0.3640`，20260206 `0.3614`
3. `300065`：20260223 `0.3735`，20260224 `0.3739`，20260225 `0.3736`
4. `000426`：20260206 `0.3609`
5. `688498`：20260205 `0.3516`

这些边缘样本的结构有两个值得注意的点：

1. `300065`、`688498` 这类样本根本没有触发会改写分数的仲裁动作，说明它们停在阈值下方主要是原始策略贡献不足，而不是仲裁把它们压下来。
2. `603993`、`300724`、`000426` 虽然带有 `short_hold` 或 `long_hold` 标记，但这两类动作在当前代码里只是持有周期提示，不直接修改 `score_b`，因此它们也不能解释为何样本没过 0.38。

换句话说，最接近阈值的这一批票，绝大多数不是被 score-mutating arbitration 打下来的，而是原始 Layer B 得分本来就没有足够越过 fast gate。

### 8.6 对后续优先级的影响

这条证据会直接影响后续排查顺序：

1. 如果只看边缘样本，优先级应先放在 Layer B 原始分数形成机制，而不是先改仲裁。
2. 仲裁仍然值得分析，但更像第二优先级，因为它解释的是一部分更低分样本，而不是 0.35 到 0.38 这批最接近过线的票。
3. 这进一步支持一个更谨慎的业务判断：在没有先拆透原始策略贡献之前，不应该直接把问题归咎于仲裁规则，也不应立即大幅下调 fast gate。

## 9. 子因子级别拆解

继续把 `0.35 <= score_b < 0.38` 的边缘样本下钻到 sub-factor 层后，问题进一步收敛。

### 9.1 event_sentiment 不是“弱”，而是大多数时间根本缺席

边缘样本的一个显著共同点是：

1. 除 `000426` 外，`event_sentiment` 的 completeness 基本都为 `0.0`
2. 这意味着不是这条策略给了弱正分或弱负分，而是它在多数边缘样本里没有形成有效输入
3. 只有 `000426` 在 20260206 通过 `insider_conviction` 提供了 `+0.05` 的增量，其余新闻相关子因子依然为 0

这说明当前 Layer B 更像是一个“缺少第三条腿”的结构：多数边缘样本只能依赖 trend 和 fundamental 两个大类，event_sentiment 没有稳定补分能力。

### 9.2 trend 内部：ema_alignment 很稳定，但 volatility 常常拖分

边缘样本中，trend 子因子呈现出相对稳定的模式：

1. `ema_alignment` 几乎总是稳定正贡献，常见值约 `+0.1125`
2. `adx_strength` 通常是小到中等正贡献
3. `momentum` 在部分股票上是强正贡献，但也有一些样本只有中性或接近 0
4. `volatility` 则经常是负项，例如 `603993` 约 `-0.075`，`300724` 约 `-0.0663`，`000426` 约 `-0.06`

这意味着 trend 本身不是简单的单边正向加分器。很多接近阈值的票，已经拿到了很强的均线排列和部分动量分，但同时被 volatility 明显往回拉。

### 9.3 fundamental 内部：profitability 和 financial_health 更稳定，growth 系列更分化

fundamental 子因子分布比表层更有信息量：

1. `profitability` 和 `financial_health` 在多数边缘样本中持续提供正贡献
2. `growth` 在不同股票之间分化明显，有的为正，有的为负，有的接近 0
3. `growth_valuation` 也不稳定，例如 `000426` 的该项为负
4. `industry_pe` 在这批边缘样本里基本长期为 0，没有形成有效区分度

代表性例子：

1. `603993`：profitability、financial_health、growth_valuation 为正，但 `growth` 为负，抵消了一部分 fundamental 总分
2. `300724`：fundamental 结构相对完整，profitability、growth、financial_health、growth_valuation 都是正项
3. `300065`：financial_health、growth 为正，但 profitability 明显为负，导致总分始终卡在阈值下方
4. `000426`：profitability 为正，但 growth 与 financial_health 基本不给增量，growth_valuation 还是负项

### 9.4 mean_reversion 基本不是主驱动项

边缘样本里，mean_reversion 大多数时候都不构成主要决定因素：

1. 很多票的 `zscore_bbands`、`rsi_extreme`、`stat_arb`、`hurst_regime` 都接近 0
2. 即便像 `300065` 这样有一点 `stat_arb` 正贡献，也会被 `hurst_regime` 的负贡献大致抵消

因此，当前窗口里的边缘样本并没有体现出“均值回归子因子明显被压制，导致整体不过线”的特征。

### 9.5 目前最有价值的业务解释

把子因子层证据和前面的策略层证据合起来，当前最有价值的解释是：

1. Layer B 的边缘样本通常已经具备较强的 trend / fundamental 主干信号
2. 但 trend 会被 volatility 回拉，fundamental 会被 growth 或 growth_valuation 的分化项削弱
3. event_sentiment 又大多直接缺席，无法提供第三个稳定增量
4. 最终使得大量样本停在 `0.35 ~ 0.38` 下沿，而不是顺利越过 fast gate

## 10. 当前业务主线的下一步

到这一步，下一轮分析的优先级已经可以再缩小：

1. 优先检查为什么边缘样本的 `event_sentiment` completeness 长期为 0
2. 同时检查 trend 的 `volatility` 负贡献是否系统性偏强
3. 再检查 fundamental 的 `growth` / `growth_valuation` 是否在某些类别股票上过于保守

这三个问题都比“直接下调 fast gate”更值得先回答。

## 11. 本次分析的定位

这份扫描的价值，不是直接给出最终调参结论，而是把业务主线进一步收敛到一个更小的问题集合：

1. 不是 provider 速度问题。
2. 不是 rolling scheduler 问题。
3. 不是 Layer C 主导的问题。
4. 当前最值得深挖的是 Layer B 分数分布、仲裁动作与 fast gate 的相互作用。
