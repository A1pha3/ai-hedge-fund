# 多因子短期动量增强策略：可实现版本

## 结论先行

把主收益窗口从 3 到 8 日改成 2 到 9 日，我认为是合理的，而且比原来的设定更接近真实交易。原因很直接：

1. 2 日下限允许策略承认“强样本可以更早兑现”，不必人为等到第 3 天。
2. 9 日上限给了主线扩散和二波延续一点空间，不会把所有强票都压缩在过窄窗口里。
3. 这不会推翻文档里的五大因子组，但会改变标签定义、部分 lookback 设计、留存因子的权重和退出合同。

真正需要明确的是，下文所有 T+N 都以实际买入日为 T0。也就是说，2 到 9 日窗口指的是从实际入场之后开始计算的持有与评估区间。

## 把时间窗口改成 2 到 9 日，是否会影响因子

会影响，但影响主要集中在标签、权重和退出规则，不是把整套因子推倒重来。

| 层面 | 影响程度 | 具体变化 |
| :--- | :--- | :--- |
| 标签定义 | 高 | 尾部标签从“3 到 8 日最高涨幅是否到 20%”改成“2 到 9 日最高涨幅是否到 20%” |
| 执行确认 | 中 | T+1 和 T+2 的强弱变得更重要，因为现在允许更早兑现 |
| 因子大类 | 低 | 五个因子组不变，仍然是板块扩散、主动资金、价格结构、注意力、可交易性/留存 |
| 因子权重 | 中 | 留存和结构的权重应略升，纯情绪权重应略降 |
| 攻击斜率公式 | 中 | 从 3 日、5 日为主，扩展为 3 日、5 日、8 日混合 |
| 涨停基因记忆 | 中 | 从近 2 到 4 日的狭义记忆，扩成近 2 日、5 日、9 日三段记忆 |
| 退出合同 | 高 | 应拆成 2 到 3 日快速兑现段、4 到 6 日主交易段、7 到 9 日尾部扩张段 |

一句话总结：2 到 9 日不会改变“该看哪些因子”，但会改变“这些因子为谁服务”。原来它们主要服务于 3 到 8 日爆发，现在它们同时服务于更早兑现和更晚扩张。

## 原稿的主要问题

| 问题 | 为什么有问题 | 直接后果 |
| :--- | :--- | :--- |
| 把 20% 极端收益当成主标签 | 20% 在 2 到 9 日窗口里依然是尾部事件，样本天然稀疏 | 很容易过拟合，回测好看但样本外失效 |
| 混用了不可比的研究指标 | Rank IC、多空年化、多头超额、指数增强收益提升来自不同市场、频率、股票池和成本假设 | 这些数字不能直接拿来排序，更不能直接拿来定权重 |
| 因子之间高度重叠 | 引力量价、大单主动流本质上都在描述资金驱动；有限关注和涨停基因都在描述情绪拥挤与股性活跃 | 静态加权会重复计分，表面分散，实际集中 |
| 没有把 A 股可交易性放进模型 | 一字板买不到、T+1 不能当日止损、涨跌停制度、不同板块涨跌幅限制都会改变真实收益分布 | 回测收益和实盘收益偏差会非常大 |
| 缺少市场状态和风格状态约束 | 短线动量因子对行情环境极其敏感，情绪退潮和高波动切换期会快速失效 | 在最不该出手的时候出手，胜率和赔率同时恶化 |
| 缺少留存质量约束 | 很多票盘中会冲高，但收盘留不住，第二天也未必有舒服的交易点 | 账面命中率不低，实际可兑现收益很差 |
| 仓位设计过于激进 | 前 3 到 5 只、单票上限 25%，对短线高波动小票太重 | 容易被单一失败样本拖穿组合 |
| 没有验证框架 | 没有样本外滚动、没有事件去重、没有滑点和成交限制、没有分板块和分行情检验 | 无法判断策略到底是真有效还是数据偶然 |

## 目标与标签重新定义

如果目标仍然写成“2 到 9 日 20% 涨幅且胜率超过 50%”，策略仍然会走偏。更合理的做法，是把目标拆成执行确认、主交易留存、尾部爆发三层。

| 层级 | 观察窗口 | 真正要解决的问题 | 建议标签 |
| :--- | :--- | :--- | :--- |
| 第一层：执行确认 | T+1 到 T+2 | 入场后能不能迅速给出可兑现的延续性 | 盘中高点触达、T+2 收盘是否仍为正 |
| 第二层：主交易留存 | T+2 到 T+4 | 强势能不能留到收盘并延续到主交易段 | T+2 到 T+4 的收盘留存率 |
| 第三层：尾部爆发 | T+2 到 T+9 | 在已经具备留存的样本里，谁有可能跑出 20% | 2 到 9 日窗口内最高涨幅是否达到 20% |

推荐把标签写成下面这样，方便后续直接落到回测和 short trade target 逻辑里：

```text
entry_price = next_day_vwap if available else next_day_open

r_high(h) = high[T+h] / entry_price - 1
r_close(h) = close[T+h] / entry_price - 1

label_fast_confirm = 1
  if max(r_high(1), r_high(2)) >= 0.04
  or r_close(2) >= 0.01
  else 0

label_retention = 1
  if sum(1 for h in [2, 3, 4] if r_close(h) > 0) >= 2
  and mean(r_close(2), r_close(3), r_close(4)) >= 0.01
  else 0

label_tail_20 = 1
  if max(r_high(h) for h in [2, 3, 4, 5, 6, 7, 8, 9]) >= 0.20
  else 0
```

这样做之后，20% 不再是唯一目标，而是高分层样本的尾部奖励。策略首先追求的是可交易延续和收盘留存，其次才是尾部爆发。

## 更好的策略框架

这个仓库本身已经有比较清晰的短线框架雏形：

1. Layer A 候选池负责过滤 ST、北交所、次新、停牌、涨停买不到和低流动性样本。
2. 市场状态模块已经能给出 aggressive_trade、normal_trade、shadow_only、halt 这类门控结果。
3. Layer B 评分器已经有趋势、事件、基本面等信号聚合能力。
4. 最终的 short trade target profile 决定正式入选、近失误和 shadow lane 的边界。

所以更优方案不应该另起炉灶，而应该在现有框架上把短线动量改成下面这条链路：

市场门控 -> 板块扩散 -> 个股资金与结构评分 -> 可交易性过滤 -> 入选与持仓合同

## 2 到 9 日窗口下的动态权重

相比之前的版本，我建议把纯情绪权重略降，把留存和结构权重略升。

| 因子组 | aggressive_trade | normal_trade | shadow_only / halt |
| :--- | :--- | :--- | :--- |
| 板块扩散与催化 | 24% | 28% | 0 |
| 主动资金与微观结构 | 31% | 24% | 0 |
| 价格结构与攻击斜率 | 21% | 22% | 0 |
| 注意力与股性 | 8% | 8% | 0 |
| 可交易性与留存 | 16% | 18% | 0 |

对应的门控系数建议写成：

```text
gate_coeff = 1.10  if gate == aggressive_trade
gate_coeff = 1.00  if gate == normal_trade
gate_coeff = 0.00  if gate in {shadow_only, halt}
```

shadow_only 和 halt 不做正式交易，只保留 shadow 跟踪样本，用来验证是否误杀。

## 可直接实现的评分协议

### 一、统一输出对象

为了对齐仓库当前实现，所有短线因子都不应该再单独定义新模型，而应该沿用现有数据结构：

1. 每个子因子输出一个 SubFactor。
2. 每个因子组通过 aggregate_sub_factors 聚合成一个 StrategySignal。
3. 最终再把五个因子组的 group_score 聚合成一个短线总分。

建议的字段映射如下：

```text
SubFactor(
  name,
  direction,      # -1, 0, 1
  confidence,     # 0 ~ 100
  completeness,   # 0 ~ 1
  weight,         # 组内权重
  metrics         # 原始指标字典
)
```

分钟级、日线级、板块级和历史先验指标都可以塞进 metrics，不需要改 CandidateStock 或 StrategySignal 的 Pydantic 结构。

### 二、统一分数映射

为了让不同来源的因子能落到同一套分数协议里，建议统一采用“原始值 -> raw_score -> direction/confidence”三步映射。

对于横截面因子，优先用鲁棒标准化：

```text
z_robust(x) = clip((x - median(x_cross_section)) / (1.4826 * MAD(x_cross_section) + 1e-6), -3, 3)
raw_score = clip(50 + 20 * z_robust(x), 0, 100)
```

对于绝对阈值因子，直接用评分表映射到 raw_score，不必先做 zscore。

统一映射规则建议为：

```text
direction = 1   if raw_score >= 60
direction = -1  if raw_score <= 40
direction = 0   otherwise

confidence = min(100, 2 * abs(raw_score - 50))
```

统一聚合规则建议为：

```text
group_score_g = sum(w_j * direction_j * confidence_j / 100 for each subfactor j in group g)

final_score = gate_coeff * sum(W_g * group_score_g for each group g) - penalty_total
```

其中：

1. `w_j` 是组内权重。
2. `W_g` 是组间权重。
3. 缺失数据时，`completeness = 0`，聚合时按当前仓库的 `aggregate_sub_factors` 逻辑自动忽略。

### 三、正式入选阈值

在 2 到 9 日窗口下，不建议只看 final_score 排名，而应该同时看 hard gate。

| Gate | formal selected | near miss | shadow only |
| :--- | :--- | :--- | :--- |
| aggressive_trade | `final_score >= 0.42` | `0.32 <= final_score < 0.42` | `< 0.32` |
| normal_trade | `final_score >= 0.46` | `0.35 <= final_score < 0.46` | `< 0.35` |
| shadow_only / halt | 不允许 | 不允许 | 全部转 shadow |

正式入选前，至少还要同时满足下面四条硬门槛：

```text
group_score_sector >= 0.05
group_score_flow   >= 0.08 if aggressive_trade else 0.05
group_score_retention >= 0.00 if aggressive_trade else 0.05
penalty_total <= 0.12
```

如果 `group_score_sector` 和 `group_score_flow` 里有任何一个为负，直接不进正式组合。

## 五个因子组的公式、阈值与评分表

### 一、板块扩散与催化

这一组解决的是“有没有方向可做”，而不是“单只股票会不会涨”。

| 子因子 | 公式 | 组内权重 | 推荐评分阈值 |
| :--- | :--- | :--- | :--- |
| `sector_amt_share` | `sector_amount_t / market_amount_t` | 0.30 | 90 分: `>= 0.06`; 75 分: `>= 0.04`; 60 分: `>= 0.025`; 40 分: `>= 0.015`; 20 分: `< 0.015` |
| `sector_breadth_3` | `count(ret_1d > 0.03 within sector) / count(valid_members)` | 0.25 | 90 分: `>= 0.35`; 75 分: `>= 0.25`; 60 分: `>= 0.18`; 40 分: `>= 0.10`; 20 分: `< 0.10` |
| `follow_ratio_2` | `count(nonleader_ret_1d > 0.02) / max(valid_nonleader, 1)` | 0.25 | 90 分: `>= 0.30`; 75 分: `>= 0.22`; 60 分: `>= 0.15`; 40 分: `>= 0.08`; 20 分: `< 0.08` |
| `catalyst_freshness` | `exp(-0.35 * days_since_positive_event)` | 0.20 | 90 分: `>= 0.70`; 75 分: `>= 0.50`; 60 分: `>= 0.35`; 40 分: `>= 0.20`; 20 分: `< 0.20` |

没有事件数据时，`catalyst_freshness.completeness = 0`，其余子因子按权重重归一即可。

### 二、主动资金与微观结构

这一组解决的是“主导资金是否真的在推”，在 2 到 9 日窗口下，它比以前更重要，因为 2 日下限意味着早期确认更关键。

| 子因子 | 公式 | 组内权重 | 推荐评分阈值 |
| :--- | :--- | :--- | :--- |
| `flow_60` | `(big_buy_amt_60 - big_sell_amt_60) / turnover_60` | 0.40 | 90 分: `>= 0.12`; 75 分: `>= 0.08`; 60 分: `>= 0.04`; 40 分: `>= 0.00`; 20 分: `< 0.00` |
| `persist_120` | `positive_net_buy_minutes_120 / active_minutes_120` | 0.30 | 90 分: `>= 0.65`; 75 分: `>= 0.58`; 60 分: `>= 0.52`; 40 分: `>= 0.45`; 20 分: `< 0.45` |
| `close_support_30` | `net_buy_amt_last30 / turnover_last30` | 0.30 | 90 分: `>= 0.10`; 75 分: `>= 0.05`; 60 分: `>= 0.02`; 40 分: `>= 0.00`; 20 分: `< 0.00` |

如果分钟级数据不全，可以退化成日度代理：

```text
flow_60           -> daily_main_net_inflow / daily_turnover
persist_120       -> positive_intraday_bar_count / all_intraday_bar_count
close_support_30  -> last_30m_turnover_share * last_30m_net_inflow_ratio
```

### 三、价格结构与攻击斜率

这一组解决的是“上涨是不是健康而不是透支”。2 到 9 日窗口下，这一组的 lookback 应该从 3 日、5 日扩展成 3 日、5 日、8 日混合。

| 子因子 | 公式 | 组内权重 | 推荐评分阈值 |
| :--- | :--- | :--- | :--- |
| `attack_slope_358` | `100 * (0.5 * slope(log(close), 3) + 0.3 * slope(log(close), 5) + 0.2 * slope(log(close), 8))` | 0.30 | 90 分: `>= 1.20`; 75 分: `>= 0.80`; 60 分: `>= 0.40`; 40 分: `>= 0.00`; 20 分: `< 0.00` |
| `breakout_quality_20_atr` | `(close_t - HHV(high, 20)_{t-1}) / ATR_14` | 0.25 | 90 分: `>= 1.00`; 75 分: `>= 0.40`; 60 分: `>= 0.00`; 40 分: `>= -0.50`; 20 分: `< -0.50` |
| `close_structure` | `CLV - 0.5 * upper_shadow_ratio` | 0.25 | 90 分: `>= 0.70`; 75 分: `>= 0.58`; 60 分: `>= 0.45`; 40 分: `>= 0.32`; 20 分: `< 0.32` |
| `amount_ratio_5` | `amount_t / MA_5(amount)` | 0.20 | 90 分: `>= 2.00`; 75 分: `>= 1.50`; 60 分: `>= 1.10`; 40 分: `>= 0.90`; 20 分: `< 0.90` |

其中：

```text
CLV = (close_t - low_t) / (high_t - low_t + 1e-6)
upper_shadow_ratio = (high_t - max(open_t, close_t)) / (high_t - low_t + 1e-6)
```

### 四、注意力与股性

这一组只负责做情绪放大，不负责独立给出买入结论。窗口改成 2 到 9 日之后，它的重要性不应上升，反而应该稍微下降。

| 子因子 | 公式 | 组内权重 | 推荐评分阈值 |
| :--- | :--- | :--- | :--- |
| `turnover_ratio_20` | `turnover_t / median(turnover_{t-20:t-1})` | 0.30 | 90 分: `>= 2.20`; 75 分: `>= 1.60`; 60 分: `>= 1.20`; 40 分: `>= 0.90`; 20 分: `< 0.90` |
| `attention_composite` | `0.4 * pct_rank(turnover_ratio_20) + 0.3 * pct_rank(amount_ratio_5) + 0.3 * pct_rank(abs(ret_3d))` | 0.35 | 90 分: `>= 0.80`; 75 分: `>= 0.65`; 60 分: `>= 0.50`; 40 分: `>= 0.35`; 20 分: `< 0.35` |
| `limit_up_memory_259` | `0.5 * I(limit_up in last 2d) + 0.3 * I(limit_up in last 5d) + 0.2 * I(limit_up in last 9d)` | 0.20 | 85 分: `>= 0.80`; 70 分: `>= 0.50`; 55 分: `>= 0.30`; 35 分: `< 0.30` |
| `dragon_tiger_bonus` | `1 if LHB_or_institutional_net_buy else 0` | 0.15 | 80 分: `1`; 50 分: `0`; 无数据则 completeness = 0 |

这一组的核心不是“越热越好”，而是“热度有增量，但不能过热”。因此过热部分应该放到 penalty，而不是继续当加分项。

### 五、可交易性与留存

这是 2 到 9 日窗口下最需要强化的一组。因为 2 日下限意味着早兑现可以成立，9 日上限又意味着收盘留存不够强的样本很难撑到后面。

| 子因子 | 公式 | 组内权重 | 推荐评分阈值 |
| :--- | :--- | :--- | :--- |
| `retention_proxy` | `0.5 * close_structure + 0.3 * (1 - upper_shadow_ratio) + 0.2 * clip((close_t / vwap_t - 1) / 0.03, 0, 1)` | 0.30 | 90 分: `>= 0.75`; 75 分: `>= 0.65`; 60 分: `>= 0.55`; 40 分: `>= 0.45`; 20 分: `< 0.45` |
| `supply_pressure_60` | `count(close_{t-k} in [close_t, 1.03 * close_t], k=1..60) / 60` | 0.20 | 90 分: `<= 0.08`; 75 分: `<= 0.12`; 60 分: `<= 0.18`; 40 分: `<= 0.25`; 20 分: `> 0.25` |
| `failed_breakout_10` | `count((high_i > HHV(high, 20)_{i-1}) and (close_i < open_i), i=t-9..t)` | 0.20 | 90 分: `0`; 75 分: `1`; 55 分: `2`; 30 分: `>= 3` |
| `prior_retention_score` | `100 * (0.45 * next_close_positive_rate + 0.35 * clip(next_open_to_close_return_mean / 0.03, 0, 1) + 0.20 * next_high_hit_rate_at_0.04)` | 0.30 | 90 分: `>= 75`; 75 分: `>= 65`; 60 分: `>= 55`; 40 分: `>= 45`; 20 分: `< 45` |

其中：

```text
vwap_t = amount_t / volume_t
```

`prior_retention_score` 如果没有历史先验，就把 completeness 设为 0，让另外三个留存子因子自动重归一。

## 惩罚项与硬过滤

为了避免注意力因子和涨停记忆把系统推向最拥挤、最难成交的位置，建议显式写出 penalty。

```text
overheat_penalty = 0.08
  if turnover_ratio_20 >= 2.5
  and limit_up_memory_259 >= 0.8
  and amount_ratio_5 >= 2.0
  else 0.04 if turnover_ratio_20 >= 2.0 and amount_ratio_5 >= 1.8 else 0.00

weak_close_penalty = 0.06
  if close_structure < 0.45 and close_support_30 < 0.02
  else 0.03 if close_structure < 0.50 else 0.00

congestion_penalty = 0.05
  if supply_pressure_60 > 0.25
  else 0.02 if supply_pressure_60 > 0.18 else 0.00

penalty_total = overheat_penalty + weak_close_penalty + congestion_penalty
```

另外，下面三种情况建议直接 hard reject，不进入 formal selected：

1. 当日收盘已经非常接近涨停，次日大概率失去舒服买点。
2. 过去 10 日失败突破次数过多，且 `close_structure` 明显偏弱。
3. 板块扩散没起来，但个股注意力和股性单独非常高。

## 推荐的交易合同

因子打分只是上半场，真正决定收益分布的是交易合同。窗口改成 2 到 9 日之后，交易合同必须分段。

### 入场原则

1. 只在 aggressive_trade 和 normal_trade 环境下允许正式开仓。
2. 只交易板块扩散明确的前 3 到 5 个方向，不做孤立个股。
3. 次日如果高开过大，需要二次确认，不能把所有高开都当成强势。
4. 如果样本高分但预计次日可交易性差，宁可进 shadow lane，也不要强行纳入正式组合。

### 持仓原则

1. aggressive_trade 环境下正式持仓 2 到 3 只。
2. normal_trade 环境下正式持仓 1 到 2 只。
3. 单票初始仓位建议控制在 8% 到 12%，单票上限不超过 15%。
4. 同一题材总暴露不超过 25%。

### 退出原则

建议把 2 到 9 日分成三段，而不是统一一个卖点：

1. 2 到 3 日快速兑现段：如果 `label_fast_confirm` 没有被满足，或者 T+1 到 T+2 明显收盘留存失败，直接减仓或退出。
2. 4 到 6 日主交易段：只有板块扩散仍在、主动资金未明显衰退、收盘留存仍强时，才继续持有主仓位。
3. 7 到 9 日尾部扩张段：只保留最后三分之一尾仓去博弈 20% 尾部，不用全部仓位硬等。
4. 20% 不再是统一卖点，而是尾部仓位的奖励目标。

## 验证框架必须补上

短线策略如果没有验证框架，基本等于没有策略。新方案至少要补上这几层验证：

### 一、样本外滚动验证

1. 按月份或季度做 walk-forward。
2. 训练窗口和测试窗口必须分离。
3. 重大事件窗口要做去重或隔离，避免同一主题被重复学习。

### 二、真实交易约束回放

1. 模拟一字板买不到。
2. 模拟涨跌停制度与不同板块涨跌幅约束。
3. 模拟 T+1 不能当日止损。
4. 把滑点、手续费和高换手冲击成本写进回测。

### 三、分环境检验

1. 分 aggressive_trade、normal_trade、shadow_only 检验。
2. 分主板、创业板、科创板检验。
3. 分高情绪扩散、低情绪退潮阶段检验。

### 四、真正该看的指标

| 指标 | 用途 |
| :--- | :--- |
| T+1 / T+2 高点触达率 | 判断有没有快速兑现机会 |
| T+2 / T+4 收盘正收益率 | 判断强势是否留到主交易段 |
| T+5 / T+9 收盘中位收益 | 判断是不是只有前两天强，后面完全断掉 |
| 2 到 9 日 20% 命中率 | 只用来评估尾部捕获，不作为唯一成功标准 |
| 交易可执行率 | 判断高分股里到底有多少真正能买到和卖好 |
| 组合回撤与暴露集中度 | 判断仓位设计是否过重 |

如果一个方案只是把 20% 的命中率做高了，但 T+2 / T+4 收盘胜率、可执行率和盈亏比都变差，那这个方案应该被直接否掉。

## 最终建议

如果你的目标是把这套方案推进到仓库实现，我建议把执行顺序固定为下面四步：

1. 先按上面的标签定义，把 2 到 9 日的 fast confirm、retention、tail 结果写进回测样本。
2. 再按五个因子组分别产出 SubFactor 和 group_score，不要一开始就直接混成总分。
3. 然后用 hard gate 和 penalty 把最拥挤、最难成交、最容易收盘失败的样本先剔掉。
4. 最后再做 threshold grid 和 walk-forward，确定 aggressive_trade 与 normal_trade 各自的正式入选阈值。

这套策略真正应该追求的，不是“所有入选样本都去冲 20%”，而是下面三件事同时成立：

1. 在适合进攻的市场环境里，能稳定找到 T+1 能确认、T+2 到 T+4 能留存的样本。
2. 这些样本在 4 到 6 日主交易段仍然具备正期望，而不是只会盘中冲高。
3. 20% 的尾部爆发更多集中在最高分层，而不是随机散落。

把窗口改成 2 到 9 日之后，文档中的因子不是失效了，而是角色更清楚了：主动资金负责早确认，价格结构和留存负责撑住中段，板块扩散决定能不能把少数样本送进 20% 的尾部区间。
