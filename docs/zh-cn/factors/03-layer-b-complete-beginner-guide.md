## 2. 先用一句话理解 Layer B

如果只允许记一句话，请记这句：

**Layer B 是一层“规则化、多因子、可批量执行”的中间筛选层。它把 Layer A 的大候选池，压缩成值得进入 Layer C 深度研究的高优先级样本。**

这句话里有四个关键词。

### 2.1 规则化

它主要靠明确规则计算，不依赖 LLM 临场发挥。

这意味着：

1. 结果可重复。
2. 速度可控。
3. 方便批量回放和反事实验证。

### 2.2 多因子

它不是只看一个指标，而是把趋势、均值回归、基本面、事件情绪四类证据放在一起。

这意味着：

1. 单一因子再强，也不应脱离整体结构理解。
2. 得分高不代表“什么都好”，只代表“在当前规则体系下综合证据较强”。

### 2.3 中间层

Layer B 不是最终买入决策。

它的下游还有：

1. Layer C 多智能体深度分析。
2. watchlist 门槛。
3. buy order 执行承接。

所以：

1. Layer B 分数高，不等于最终一定买。
2. Layer B 问题，和最终没成交，不一定是一回事。

### 2.4 高优先级压缩器

在产品设计里，Layer A 候选池通常是 150 到 300 只左右，Layer B 的目标是把它压缩到约 60 只高分池，再进入 Layer C。

这一步的本质不是“直接选赢家”，而是“把更值得花昂贵研究资源的票先挑出来”。

## 3. Layer B 在整个流水线里的位置

先把全流程看清楚，后面每一个细节才不会理解歪。

```text
Layer A 候选池
  -> Layer B 四策略评分与融合
  -> Layer C 多智能体深度分析
  -> watchlist
  -> buy_orders
  -> execution
```

对应理解如下：

1. Layer A 负责“先把明显不该看的票排掉”。
2. Layer B 负责“用规则和因子把值得研究的票排出优先级”。
3. Layer C 负责“让 18 个 Agent 做更贵、更深的定性与综合分析”。
4. execution 负责“研究上看起来可以，不代表交易上今天一定能做”。

因此，Layer B 的定位很像一道前置研究闸门。

它不是：

1. 最终投资结论。
2. 真实成交引擎。
3. LLM 主导的智能判断层。

它是：

1. 高吞吐的规则打分层。
2. 研究资源分配层。
3. 后续深度分析的上游过滤器。

## 4. Layer B 的核心数据结构

如果你是新手，建议先记住三个对象，而不是一上来记全部函数。

这些对象定义在 [src/screening/models.py](../../src/screening/models.py)。

### 4.1 SubFactor：最小因子颗粒

每个子因子至少有五个核心字段：

1. `name`：因子名。
2. `direction`：方向，取值为 `-1`、`0`、`1`。
3. `confidence`：置信度，范围 `0 ~ 100`。
4. `completeness`：数据完整度，范围 `0 ~ 1`。
5. `weight`：该子因子在所属策略内部的权重。

你可以把它理解成：

“某条策略里的一个小证据片段”。

### 4.2 StrategySignal：单条策略的输出三元组

每条策略最终都会被压缩成一个统一格式：

1. `direction`
2. `confidence`
3. `completeness`

再带上一份 `sub_factors` 明细，方便解释来源。

这一步很关键，因为它把四条不同风格的策略，统一成后续可融合的标准接口。

### 4.3 FusedScore：最终 Layer B 融合结果

融合后输出的核心对象有：

1. `score_b`：Layer B 综合得分，范围 `-1 ~ 1`。
2. `strategy_signals`：四条策略的标准化输出。
3. `arbitration_applied`：仲裁规则命中记录。
4. `weights_used`：最终实际使用的权重。
5. `decision`：基于 `score_b` 的 Layer B 决策标签。

也就是说，Layer B 不只是吐出一个数字，它还会吐出“这个数字怎么来的”。

## 5. 四条策略到底在看什么

Layer B 的主干在 [src/screening/strategy_scorer.py](../../src/screening/strategy_scorer.py)。

可以粗略理解为四个问题：

1. 趋势是不是在增强。
2. 是否存在均值回归机会。
3. 基本面是不是足够撑得住。
4. 最近事件和情绪是不是在加分或减分。

下面逐条拆开。

### 5.1 策略一：趋势跟踪

趋势策略主要回答：

**这只票当前是不是处于一个较健康的顺势结构里。**

它现在默认包含五个子因子。

#### 5.1.1 ema_alignment

逻辑：

1. 计算 `EMA10`、`EMA30`、`EMA60`。
2. 如果 `EMA10 > EMA30 > EMA60`，视为多头排列。
3. 如果 `EMA10 < EMA30 < EMA60`，视为空头排列。
4. 否则视为中性。

置信度来源：

1. 不是人工拍脑袋给的。
2. 而是看几条均线彼此的间距占当前价格的比例。
3. 均线拉得越开，趋势表达越清晰，置信度越高。

你可以把它理解成：

“趋势有没有排好队”。

#### 5.1.1A long_trend_alignment

这是后来补进来的长期趋势确认因子，默认开启。

逻辑：

1. 计算 `EMA10` 和 `EMA200`。
2. 如果 `EMA10 > EMA200`，视为长期多头背景成立。
3. 如果 `EMA10 < EMA200`，视为长期空头背景成立。
4. 如果历史数据不足 `200` 根K线，这个因子直接记为不可用，不参与聚合。

它和 `ema_alignment` 的分工不同：

1. `ema_alignment` 看的是短中期结构有没有排好队。
2. `long_trend_alignment` 看的是更大的长期背景是否已经转强。
3. 这两个因子并列存在，不是互相替代。

置信度来源：

1. 不是看 `EMA10` 与 `EMA30`、`EMA60` 的双间距。
2. 而是只看 `EMA10` 与 `EMA200` 之间的距离占当前价格的比例。
3. 距离越大，说明长期趋势背景越清晰，置信度越高。

#### 5.1.2 adx_strength

逻辑：

1. 先算 `ADX`。
2. 如果 `ADX < 20`，说明趋势强度不够，方向直接记为中性。
3. 如果 `ADX >= 20`，再比较 `+DI` 和 `-DI`，决定偏多还是偏空。

这一步不是在看涨跌本身，而是在看“趋势是不是足够成型”。

#### 5.1.3 momentum

这部分复用了技术分析 Agent 的动量信号函数。

你可以把它理解成：

1. 趋势不是只看均线排布。
2. 还要看近期上涨或下跌的动能是否延续。

#### 5.1.4 volatility

波动率子因子并不是简单把“波动越大越好”当正面。

它更像是在判断：

1. 当前波动结构是否支持趋势延续。
2. 还是已经乱到不适合继续追随。

#### 5.1.5 趋势策略内部权重

默认权重是：

1. `ema_alignment = 0.26`
2. `adx_strength = 0.21`
3. `momentum = 0.21`
4. `volatility = 0.17`
5. `long_trend_alignment = 0.15`

这反映的思想是：

1. 短中期均线结构和趋势强度仍然是主干。
2. 动量与波动继续负责辅助确认。
3. 新增的长期趋势因子负责补充更大级别的背景过滤。

#### 5.1.6 默认开启后的第一次窗口验证怎么看

当前主线已经把 `long_trend_alignment` 设成默认开启，但这不代表它一定会立刻放出更多 Layer B 样本。

在 `20260302 .. 20260326` 的 19 个交易日窗口里，默认开启版与显式关闭版对比结果是：

1. 两边的 Layer B 总通过数都等于 `1`。
2. 没有新增样本，也没有减少样本。
3. 只有 `20260325` 这一天出现了 1 个过线结果，而且开关前后完全一致。

这说明：

1. 这个新因子在当前窗口里没有制造放量副作用。
2. 它更像是一个长期背景确认因子，而不是“只要一开就会多选很多票”的阀门。
3. 因此把它默认打开，目前更像是补充趋势解释，而不是大规模改变筛选风格。

### 5.2 策略二：均值回归

均值回归策略主要回答：

**价格是不是偏离得太多，以至于存在“向均值拉回”的机会。**

它包含四个子因子。

#### 5.2.1 zscore_bbands

这部分来自 `calculate_mean_reversion_signals()`。

大意是：

1. 看价格相对均值偏离了多少。
2. 看这种偏离是否接近布林带极值区。
3. 偏离越极端，越可能触发回归信号。

#### 5.2.2 rsi_extreme

逻辑比较直白：

1. 同时看 `RSI14` 和 `RSI28`。
2. 若短中周期都过度超卖，则偏向看涨回归。
3. 若短中周期都过度超买，则偏向看跌回归。
4. 否则为中性。

关键点在于：

1. 它不是只看一个 RSI。
2. 而是用双周期减少单点噪声。

#### 5.2.3 stat_arb

这是统计套利风格的回归信号。

可以把它理解为：

1. 价格是否偏离了某种相对稳定的统计关系。
2. 如果偏离过大，是否存在回归空间。

#### 5.2.4 hurst_regime

这是均值回归策略里非常重要，也最容易被忽略的部分。

逻辑大致是：

1. 先算 Hurst 指数。
2. 若 `Hurst < 0.45`，说明更接近均值回归市场，回归信号可信度提高。
3. 若 `Hurst > 0.55`，说明更接近趋势市场，回归信号要谨慎解读。
4. 若中间区间，说明制度不清晰，方向多半给中性。

这也是后面趋势与均值回归冲突时的重要裁判。

#### 5.2.5 均值回归策略内部权重

默认权重是：

1. `zscore_bbands = 0.35`
2. `rsi_extreme = 0.20`
3. `stat_arb = 0.25`
4. `hurst_regime = 0.20`

### 5.3 策略三：基本面与估值

这条策略主要回答：

**这只票有没有足够的经营质量、成长质量和估值支撑。**

它包含五个子因子。

#### 5.3.1 profitability

这是当前最需要认真理解的一个点。

它检查三项：

1. `ROE >= 0.15`
2. `net_margin >= 0.20`
3. `operating_margin >= 0.15`

再根据达标数量做判断：

1. 至少 2 项达标：方向为正。
2. 1 项达标：方向为中性。
3. 0 项达标：默认方向为负。

这里要特别注意：

1. 这不是连续扣分，而是带有“硬 cliff”特征的规则。
2. 也就是 0 项达标时，不是“差一点”，而是直接进入强负逻辑。

这就是为什么后面很多 Layer B 讨论都围绕 profitability 展开。

#### 5.3.2 growth

这部分通过 `analyze_growth_trends()` 看连续财务期的增长质量。

重点不是“某一季增长快不快”，而是：

1. 增长是否持续。
2. 增长结构是否稳定。
3. 增长是不是足够支撑正面结论。

#### 5.3.3 financial_health

这部分通过 `check_financial_health()` 看财务健康度。

简单理解就是：

1. 盈利之外，财务结构稳不稳。
2. 有没有明显的质量风险。

#### 5.3.4 growth_valuation

这部分试图回答：

**增长值不值得当前估值。**

也就是不是只看“贵不贵”，而是看“这个贵有没有被增长合理解释”。

#### 5.3.5 industry_pe

这一步会把个股 PE 和行业中位数做比较。

大意是：

1. 比行业便宜很多，可能加分。
2. 比行业贵太多，可能减分。
3. 如果缺行业中位数或当前 PE，就不给完整度。

#### 5.3.6 基本面策略内部权重

默认权重是：

1. `profitability = 0.25`
2. `growth = 0.25`
3. `financial_health = 0.20`
4. `growth_valuation = 0.15`
5. `industry_pe = 0.15`

这反映出：

1. 盈利能力和成长质量是主干。
2. 财务健康是关键安全项。
3. 估值相关项提供辅助校正。

### 5.4 策略四：事件与情绪

这条策略主要回答：

**近期信息冲击、新闻情绪和内部人行为，是不是在给这只票加分或减分。**

它包含三个子因子。

#### 5.4.1 news_sentiment

实现方式不是让 LLM 临时写长文，而是做一层较轻的规则化聚合：

1. 把新闻标题和内容转成文本。
2. 用正负关键词做粗粒度命中。
3. 对最近 20 条新闻按时间衰减加权。
4. 最终得到方向和强弱。

这里不是“完美理解新闻”，而是“给 Layer B 一份可批量执行的事件情绪近似量”。

#### 5.4.2 insider_conviction

通过内部人交易行为看高管净买入或净卖出倾向。

直觉上：

1. 连续净买入更偏正。
2. 连续净卖出更偏负。

#### 5.4.3 event_freshness

它关心的不是“新闻多不多”，而是“最近是不是刚发生了一个有效事件”。

时间衰减函数在代码里是：

$$
w(t) = e^{-0.35t}
$$

这里的 $t$ 是事件距离当前交易日的自然日数。

这意味着：

1. 事件越新，影响越大。
2. 旧新闻即便方向明确，也会快速衰减。

#### 5.4.4 事件情绪策略内部权重

默认权重是：

1. `news_sentiment = 0.55`
2. `insider_conviction = 0.25`
3. `event_freshness = 0.20`

这说明当前事件情绪主干仍然是新闻侧，而不是内部人交易侧。

## 6. 单条策略内部是怎么聚合的

这是理解 Layer B 的关键中的关键。

相关实现见 [src/screening/strategy_scorer.py](../../src/screening/strategy_scorer.py)。

### 6.1 第一步：只让 completeness 大于 0 的子因子参与聚合

这表示：

1. 没有数据的子因子，不应该占着位置影响结果。
2. 但只要完整度大于 0，它就会进入活跃集合。

### 6.2 第二步：对子因子权重重新归一化

不是拿原始权重直接算，而是：

1. 只保留可用子因子。
2. 把这些可用子因子的权重重新归一化到总和为 1。

这一步非常重要，因为它意味着：

1. 某个因子缺失时，其余因子的相对影响会自动上升。
2. 你改一个因子的可用性，可能会改变整条策略的结构，而不是只改它自己。

### 6.3 第三步：先算方向分数

近似可理解为：

$$
score_{strategy} = \sum_{k} w_k \times direction_k \times \frac{confidence_k}{100}
$$

然后：

1. 大于 0，整条策略方向为正。
2. 小于 0，整条策略方向为负。
3. 等于 0，方向为中性。

### 6.4 第四步：再算一致性 consistency

代码不是直接把子因子置信度平均一下就结束。

它还会统计：

1. 有多少子因子和最终方向一致。
2. 一致比例越高，说明这条策略表达越清晰。

因此，策略置信度近似是：

$$
confidence_{strategy} \approx \left(\sum_k w_k \times confidence_k\right) \times consistency
$$

这也是很多直觉调参失效的根源之一。

因为：

1. 你以为只是在调一个子因子的强弱。
2. 实际上你可能在改变整条策略的方向一致性。
3. 最终让整条策略置信度一起变化。

### 6.5 第五步：再算完整度 completeness

完整度不是简单平均，而是：

1. 只对可用子因子做加权。
2. 得到一个 `0 ~ 1` 之间的完整度。

可以把它理解为：

“这条策略当前的证据链完整不完整。”

## 7. 四条策略是怎么融合成 Layer B 总分的

相关实现见 [src/screening/signal_fusion.py](../../src/screening/signal_fusion.py)。

### 7.1 标准融合公式

产品文档中的标准公式是：

$$
Score_B = \sum_i w_i \times direction_i \times confidence_i \times completeness_i
$$

在代码里，`confidence` 实际按 `0 ~ 100` 存储，计算时会再除以 `100`。

也就是说，真正含义是：

1. 方向决定正负。
2. 置信度决定强弱。
3. 完整度决定“这条信号值不值得被充分相信”。
4. 权重决定不同策略在当前环境下的话语权。

### 7.2 默认基准权重

默认基准权重定义在 [src/screening/models.py](../../src/screening/models.py)：

1. `trend = 0.30`
2. `mean_reversion = 0.20`
3. `fundamental = 0.30`
4. `event_sentiment = 0.20`

直觉解释：

1. 趋势和基本面是双主轴。
2. 均值回归和事件情绪是辅助轴。

### 7.3 不是所有策略都必然参与归一化

这里只有一个新手很容易踩的坑：

**Layer B 看的是“是否 active”，而不只是“数值大小”。**

当前逻辑里，只要某条策略 `completeness > 0`，它就会进入 active 集合，参与权重归一化。

这意味着：

1. `completeness = 1.0` 和 `completeness = 0.25` 在“能不能参与归一化”这个层面是一样的。
2. 它们的差异主要体现在最终乘到得分时的衰减，而不是 active 身份本身。

这就是为什么“把中性均值回归的完整度从 1.0 改到 0.5”常常没有你想的那么大效果。

## 8. 市场状态为什么会改权重

相关实现见 [src/screening/market_state.py](../../src/screening/market_state.py)。

Layer B 不是永远死板地用同一组权重。

原因很简单：

1. 趋势市更适合让趋势信号占优。
2. 震荡市更适合让均值回归信号占优。
3. 缩量市即便有信号，也应该更谨慎。

### 8.1 市场状态主要看哪些指标

代码当前会看：

1. 沪深 300 的 `ADX`
2. `ATR / Price`
3. 涨停跌停家数比
4. 全市场成交额
5. 北向资金连续流入或流出天数

### 8.2 当前代码里的主要判定逻辑

#### 8.2.1 Crisis

如果：

1. 指数单日跌幅很大，或
2. 跌停家数爆炸

则进入危机状态。

后果：

1. 仓位缩到 `0.3`
2. 基本面权重上调
3. 趋势权重下调

#### 8.2.2 Trend

如果：

1. `ADX > 30`
2. 同时 `ATR / Price < 0.012`

则判定为趋势状态。

后果大致是：

1. 增加趋势权重。
2. 降低均值回归权重。
3. 略降事件情绪权重。

#### 8.2.3 Range

如果：

1. `ATR / Price < 0.012`
2. 同时 `ADX < 25`

则更像震荡市。

后果大致是：

1. 提高均值回归权重。
2. 降低趋势权重。
3. 略降基本面权重。

### 8.3 成交额和北向资金的作用

#### 8.3.1 成交额过低

不是简单改权重，而是直接把 `position_scale` 降到 `0.5`。

这体现的是：

1. 市场太冷时，问题不只是“哪条策略更准”。
2. 更是“即便看起来有机会，也不该下太重”。

#### 8.3.2 北向资金连流入或连流出

代码会做额外的轻微权重修正。

这部分和产品文档不完全一样，说明实现层是“设计原则 + 落地校准”的组合，而不是一字不差照抄方案。

这是你读代码时要有的正常预期。

## 9. 冲突仲裁不是可选项，而是 Layer B 的核心

很多新手会把 Layer B 理解成：

“四条策略各自打分，然后直接加起来。”

这只说对了一半。

真正关键的是：

**Layer B 在融合前后还有一层冲突仲裁。**

### 9.1 质量优先红旗

当前代码里有一个非常重要的实际规则：`_has_quality_first_red_flag()`。

它不是产品文档里的抽象描述，而是落到了代码细节：

1. 如果 `profitability` 和 `financial_health` 同时明显偏负，且置信度较高。
2. 或者 profitability 命中“0 项达标”的硬 cliff，且没有被增长等项有效对冲。
3. 那么即便趋势和事件面看起来不错，也可能直接触发 `avoid`。

触发后会发生什么：

1. `score_b` 直接被强制为 `-1.0`
2. 决策直接标为 `strong_sell`
3. 这只票进入冷却期

这是一个很典型的“安全优先”实现。

### 9.2 强负 + 基本面负面时直接回避

如果基本面方向为负，同时存在高置信度强负信号，也会直接触发 `avoid`。

这在逻辑上表示：

1. 如果经营质量本身在塌。
2. 同时市场侧又明显不支持。
3. 那就没必要因为个别正面因子去赌。

### 9.3 短持有还是长持有

代码还会根据各策略的绝对贡献，判断更像：

1. 短持有逻辑
2. 长持有逻辑

如果短周期信号占绝对贡献 60% 以上，会打上 `short_hold`。

如果长期基本面信号占绝对贡献 60% 以上，会打上 `long_hold`。

这一步不直接改成买卖结论，但会留下持有倾向提示。

### 9.4 趋势和均值回归打架时怎么办

这一步极重要。

当趋势和均值回归同时发出反向非中性信号时：

1. 如果 Hurst 明显偏趋势，削弱均值回归置信度。
2. 如果 Hurst 明显偏回归，削弱趋势置信度。
3. 如果 Hurst 说不清，双方都降权 50%。

这其实是在说：

1. 不是每次信号冲突都该硬判一边正确。
2. 有时候最诚实的做法就是“双方都少信一点”。

### 9.5 共识加成

如果 3 条及以上策略方向一致，且强度足够，会打上 `consensus_bonus`。

后果是：

1. `score_b` 额外乘以 `1.15`
2. 但仍然限制在 `-1 ~ 1`

这体现的是：

**强共识本身也是一条信息。**

### 9.6 冷却期与提前释放

一旦因为 `avoid` 被打入冷却期，默认会进入一段时间的回避状态。

但代码并不是死锁。

如果：

1. 基本面转正，且
2. 已经满足最低冷却天数

则允许提前释放。

这说明系统的态度不是“永不再看”，而是“先冷静，等证据修复再回来”。

## 10. Layer B 的决策阈值怎么读

当前 `FusedScore.classify_decision()` 里的阈值是：

1. `score_b > 0.50`：`strong_buy`
2. `0.35 <= score_b <= 0.50`：`watch`
3. `-0.20 <= score_b < 0.35`：`neutral`
4. `-0.50 <= score_b < -0.20`：`sell`
5. `< -0.50`：`strong_sell`

要注意三件事。

### 10.1 这是 Layer B 阶段阈值，不是最终交易阈值

也就是说：

1. 它决定的是“值不值得继续研究”。
2. 不决定“今天一定下单”。

### 10.2 `watch` 的真实含义更像“研究优先级高”

很多新手会把 `watch` 理解成“快买了”。

更准确的理解是：

1. 它说明 Layer B 认为这只票有进一步研究价值。
2. 但真正能不能继续过线，还要看 Layer C 和下游闸门。

### 10.3 `strong_buy` 也会被下游否掉

因为最终还有：

$$
Score_{final} = 0.4 \times Score_B + 0.6 \times Score_C
$$

所以：

1. Layer B 只是提供量化宽度。
2. Layer C 提供智能深度。
3. 二者冲突时，最终结果可以明显不同于 Layer B 的第一印象。

## 11. 代码里还有一层常被忽略的批量计算优化

如果你只看产品文档，很容易漏掉实际运行性能设计。

在 [src/screening/strategy_scorer.py](../../src/screening/strategy_scorer.py) 里，Layer B 不是对 200 只票一视同仁地做最重计算。

### 11.1 先轻算，再重算

实际批量流程是：

1. 先对较大批次候选做轻量技术评分。
2. 根据轻量 provisional score 排序。
3. 只对前面一部分票进一步计算基本面。
4. 再只对其中更少的一部分计算事件情绪。

这是一种很典型的“粗筛后精算”结构。

### 11.2 当前相关上限参数

默认环境变量大致是：

1. `SCORE_BATCH_TECHNICAL_MAX_CANDIDATES = 160`
2. `SCORE_BATCH_FUNDAMENTAL_MAX_CANDIDATES = 100`
3. `SCORE_BATCH_EVENT_SENTIMENT_MAX_CANDIDATES = 40`
4. `SCORE_BATCH_MIN_PROVISIONAL_SCORE = 0.05`

这说明系统考虑的是：

1. 不是所有候选都值得付出同样的计算成本。
2. Layer B 自己也在做资源调度。

### 11.3 为什么这对研究也重要

因为你在回放或复盘时看到某些票没有事件情绪数据，不一定是数据出错，也可能是：

1. 它在前面的 provisional ranking 就不够靠前。
2. 没有被送进更重的阶段。

## 12. 为什么有些直觉调参会失效

这是 Layer B 最值得新手反复读的一节。

### 12.1 误区一：降低某个负项 confidence，就等于把它温和化

不对。

因为在单策略聚合里，`confidence` 不只是音量旋钮，它还会通过一致性语义影响整条策略的输出形态。

所以：

1. 你以为在减轻惩罚。
2. 结果可能是在破坏整条策略的方向表达。

### 12.2 误区二：把中性策略 completeness 降到 0.5，就等于只占半份席位

也不对。

因为在 Layer B active 归一化逻辑里，只要 `completeness > 0`，它就还是 active。

所以：

1. `0.5` 和 `1.0` 在“是否参与归一化”上没有本质区别。
2. 真正的区别只体现在最终得分乘法上。

### 12.3 误区三：Layer B 过线更多，就一定更好

也不对。

Layer B 放出更多票，可能代表三种完全不同的事情：

1. 真释放了被误伤的边缘优质样本。
2. 只是把某类结构化噪声一起放大了。
3. 增加了 Layer C 的负担，但最终并没有改善交易结果。

后面实验里你会看到，这三种事都真实发生过。

## 13. 过去这轮 Layer B 实验到底验证了什么

这一部分建议结合 [Layer B 最小规则变体验证](../analysis/layer-b-rule-variant-validation-20260312.md) 阅读。

### 13.1 profitability 的最小改动实验

研究问题是：

**当 profitability 走到“0 项达标”的硬负分支时，是否过于严厉。**

关键结论是：

1. 如果只是把它改成 active-neutral，效果并不好。
2. 更稳的分析语义是：`direction = 0`、`confidence = 0`、`completeness = 0`。
3. 也就是“温和化，但不继续占聚合席位”。

在当时窗口里，这个改动：

1. 只多放出少量边缘样本。
2. 主要集中在 `300065` 这类 hard cliff 样本。
3. 风险相对可控。

### 13.2 neutral mean_reversion 排除实验

研究问题是：

**当均值回归方向为中性时，它是否在无谓地稀释趋势和基本面双主腿。**

结果是：

1. 如果粗暴完全排除，中间层通过数会暴涨。
2. 说明这个杠杆非常敏感。
3. 也说明很多样本原本是被中性均值回归“稀释掉”的。

问题在于：

1. 放出的量太大。
2. 结构集中在 trend + fundamental 双腿、event_sentiment 缺失的样本。
3. 风险明显高于 profitability 单改。

### 13.3 guarded 模式为什么出现

因为粗暴全排除太激进，于是后续才出现 guarded 模式。

比如：

1. 只在双主腿已经足够强时才允许排除中性均值回归。
2. 并且在 profitability hard cliff 情形下仍保持保守。

这说明项目对 Layer B 的态度不是：

1. 找到一个能放出票的开关就上线。

而是：

1. 即便方向看起来对，也要控制释放量和样本结构。

### 13.4 为什么这些 Layer B 实验没有自动成为默认规则

因为后续端到端回测发现：

1. Layer B 放出的中间候选确实更多了。
2. 但很多新增样本在 watchlist 或 Layer C 那里继续被压掉。
3. 最终交易结果和收益并没有同步改善。

这点很重要。

它说明：

1. “Layer B 稍微松一点”不等于“最终系统更赚钱”。
2. 至少在那轮样本里，主要瓶颈已经不全在 Layer B。

## 14. 真实例子怎么读

看一眼真实 selection review 会很有帮助。

例如某日样本里，`600988` 的 Layer B 摘要类似：

1. `fundamental: direction = 1.00, confidence = 60.00`
2. `trend: direction = 1.00, confidence = 32.06`
3. `event_sentiment: direction = 1.00, confidence = 21.91`
4. `Layer B 综合分数为 0.3924`

这应该怎么翻译成研究语言。

### 14.1 第一层翻译：不是随机入选

这说明它不是因为某个字段偶然有值而进来，而是：

1. 基本面给了正向主支撑。
2. 趋势和事件也在同方向辅助。
3. 所以最终 `score_b` 站上了观察区间。

### 14.2 第二层翻译：主因子是谁

从强度看，最主要支撑来自基本面。

这通常比“纯事件把分冲高”更稳一些。

### 14.3 第三层翻译：还要问什么

即便如此，也仍要继续追问：

1. 事件情绪是不是只是短期噪声。
2. Layer C 是否认可这套逻辑。
3. 最终执行层能不能承接。

这就是一条合格的 Layer B 阅读路径。

### 14.4 在 Replay Artifacts 页面里，Layer B 应该看哪里

如果你是在实际工作台里读 Layer B，而不是只看源码，建议按下面顺序读。

第一步，看 `selection_review.md` 里的 Layer B 因子摘要。

这里最适合快速回答：

1. 它为什么被选出来。
2. 主支撑来自哪条策略。
3. 这份解释是原生证据，还是兼容回退。

第二步，看 `selection_snapshot.json` 或页面结构化详情里的 `layer_b_summary`。

这里最适合进一步确认：

1. `top_factors` 到底是哪几项。
2. `explanation_source` 是什么。
3. 有没有 `fallback_used`。

第三步，再看 `funnel_diagnostics.filters.layer_b`。

这里最适合回答：

1. 这天候选少，是不是大多数票在 Layer B 就被过滤掉。
2. 过滤主因是什么，例如 `below_fast_score_threshold`。
3. 当前问题更像个股判断，还是整层闸门过紧。

第四步，最后再和 Layer C、watchlist、buy order bridge 对照。

只有走完这四步，你才不会把下面几种事混为一谈：

1. Layer B 本身没放行。
2. Layer B 放行了，但 Layer C 否决。
3. Layer B 和 Layer C 都放了，但 execution 没承接。

### 14.5 解释可信度怎么分级

新手最容易忽略的一点是：

**不是所有 Layer B 解释的证据等级都一样。**

当前更稳的理解方式，是把它分成三档。

#### 14.5.1 A 档：原生 strategy_signals 解释

这类解释最强。

它通常意味着：

1. 当前运行链路里保留了原生 `strategy_signals`。
2. `selection_review.md` 和 `selection_snapshot.json` 中的 Layer B 摘要，是从真实策略信号直接透传或派生出来的。

这类证据最适合做：

1. 具体因子分析。
2. 规则语义讨论。
3. 候选之间的细粒度比较。

#### 14.5.2 B 档：基于原生信号的渲染摘要

这类解释本质上仍可靠，但它已经是“面向人阅读的摘要”，不是底层原始对象本身。

适合理解为：

1. 它适合复盘、培训和日常研究。
2. 但如果你要做精细排障，仍然应回到底层 snapshot 或信号字段。

#### 14.5.3 C 档：兼容回退解释

这是历史 frozen replay 常见情况。

如果你看到：

1. `explanation_source = legacy_plan_fields`
2. `fallback_used = true`

通常意味着：

1. 这份 Layer B 摘要不是从原生 `strategy_signals` 直接来的。
2. 而是根据 `plan.logic_scores`、`strategy_weights` 或 `adjusted_weights` 近似重建。

这类解释并不是没用，但使用方式必须保守：

1. 它适合做方向性辅助判断。
2. 不适合当成最高精度的策略证据。
3. 如果你要下“是哪条规则错了”的结论，优先回到新源或原始 plan 字段复核。

### 14.6 为什么要强调原生解释和兼容解释的区别

因为这会直接影响你能不能下强结论。

例如下面两句话，看起来很像，但证据强度完全不同：

1. “这只票的 Layer B 主因子是 fundamental 和 trend，说明它在质量和趋势上同步占优。”
2. “这只票的 Layer B 看起来像是 fundamental 和 trend 主导，但当前解释来自 legacy plan fields，结论需要保守。”

第一句更适合原生信号场景。

第二句更适合回退解释场景。

如果你把这两种情况写成同样强度的话，后续就很容易把：

1. 辅助线索误写成确定证据。
2. 历史兼容解释误写成原始策略结论。

### 14.7 给新手的最小实战练习

如果你已经把前面都看完，建议马上做一个 15 到 30 分钟的小练习。

练习步骤如下：

1. 打开一份非空 watchlist 的 `selection_review.md`。
2. 只根据 Layer B 摘要，先写一句“它为什么被选出来”。
3. 再去看 `layer_b_summary` 的 `explanation_source` 和 `fallback_used`。
4. 给这份解释判一个证据等级：A、B 或 C。
5. 再去看 Layer C 和 execution bridge，补一句“为什么它最终成交或未成交”。

如果你能稳定写出下面这种两句话结论，就说明已经真的入门：

1. “Layer B 视角下，这只票主要由 fundamental 和 trend 支撑，属于规则层面较完整的候选。”
2. “但最终是否该归因为选股有效，还要看 Layer C 共识和 execution 承接；若当前解释来自 fallback，则这一判断只应视为辅助线索。”

## 15. 新手最常见的 10 个误解

1. Layer B 分高，就等于该买。
2. Layer B 分低，就一定是这只票差。
3. 只要通过数变多，规则就优化成功。
4. 降低 confidence 一定能温和化负项。
5. 降低 completeness 一定能明显减小 active 权重。
6. 事件情绪缺失，就说明系统坏了。
7. 基本面方向为正，就不可能被 `avoid`。
8. `watch` 意味着很快下单。
9. 产品文档里的表述会和实现一字不差。
10. Layer B 与 Layer C 谁更重要，可以脱离具体窗口单独讨论。

更准确的理解是：

1. Layer B 是上游研究优先级层。
2. Layer B 的变化必须放到整条漏斗里看。
3. 文档、代码、实验、复盘要一起读，单看任何一份都可能误判。

## 18. 一页总结

最后把整份文档压成一页口径。

### 18.1 Layer B 是什么

Layer B 是规则化、多因子、中间筛选层，用来决定哪些候选值得进入更贵的 Layer C 深度分析。

### 18.2 Layer B 怎么算

1. 四条策略先各自聚合子因子。
2. 再根据市场状态调权。
3. 再做冲突仲裁。
4. 最后得到 `score_b` 和 Layer B 决策。

### 18.3 最该注意什么

1. 它不是最终买入层。
2. `confidence` 和 `completeness` 不是随便调的小旋钮。
3. 通过数变多，不代表默认规则更优。
4. 读 Layer B 时一定要同时看样本结构和下游漏斗。

### 18.4 当前这条主业务线学到了什么

1. profitability hard cliff 是真实问题，但要用对语义处理方式。
2. neutral mean_reversion 的 active 身份很敏感，不能粗暴全排。
3. 很多新增 Layer B 样本最终死在 Layer C 或 watchlist，不代表 Layer B 单独放宽就能改善最终结果。
