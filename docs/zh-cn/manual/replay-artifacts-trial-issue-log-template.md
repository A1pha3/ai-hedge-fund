# Replay Artifacts 试用问题记录模板

> 用途：
>
> 1. 在试用窗口内统一记录问题。
> 2. 把问题明确分流到“阻断缺陷”或“第二阶段治理项”。
> 3. 避免试用结束后只剩口头印象，没有可执行结论。

建议使用方式：每个问题单独复制一份模板，或在同一文档里按编号连续记录。

## 最小示例

### 1. 基本信息

- 记录编号：RA-TRIAL-001
- 记录日期：2026-03-26
- 记录人：einstein
- 试用窗口：2026-03-26 ~ 2026-04-02
- 对应 report_name：paper_trading_window_recent
- 对应 trade_date：2026-03-23
- 相关角色：周度汇总人

### 2. 问题分类

- 分类级别：B 高摩擦治理项
- 影响范围：团队普遍
- 影响动作：queue ownership、周度复盘

### 3. 问题描述

- 现象：可以认领样本，但很难快速找出即将超时或长时间无人认领的条目。
- 复现步骤：
  1. 打开 Cross-Report Workflow Queue。
  2. 切换到 unassigned。
  3. 观察当前列表只能按基础过滤查看。
- 期望结果：可以快速识别需要优先处理的旧条目。
- 实际结果：只能人工逐条判断。

### 4. 影响判断

- 是否阻断当前试用：否
- 是否有临时绕过方式：有
- 如果有，绕过方式是什么：人工按 latest_feedback_created_at 粗略判断。
- 对效率的影响：中

### 5. 初步归因

- 更像是：协作机制不足
- 补充说明：属于治理层能力缺口，不影响当前工作台基本可用性。

### 6. 归档去向

- 处理建议：纳入 P0 治理 backlog
- 建议优先级：中
- 建议负责人：后续治理阶段统一安排

### 7. 验收补充

- 是否已有截图 / 日志 / 接口证据：无
- 关联文档或路径：[docs/zh-cn/manual/replay-artifacts-trial-acceptance-plan.md](docs/zh-cn/manual/replay-artifacts-trial-acceptance-plan.md)
- 是否需要在周会复盘中单独讨论：是
- 最终结论：不阻断当前试用，但如果两周以上真实使用中持续反复出现，应进入 P0。
