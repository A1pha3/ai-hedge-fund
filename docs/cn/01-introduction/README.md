# 项目介绍类索引

这一类文档回答三个问题：**这套系统是什么、它不是什么、它为什么这么设计**。

读完这一类，你应该能判断本仓库是否解决你的问题，并理解 BTST 启用 / OversoldBounce 暂停 / 止损是摆设等「反直觉」决策背后的依据。

## 阅读顺序

**新读者**：按 1 → 2 → 3 顺序读。第一篇给全景，第二篇划边界，第三篇解释反直觉决策的依据。

**只想跑通命令的工程师**：读 1 即可，跳到 [用户手册](../02-user-manual/README.md)。

**想理解 setup 设计动机的研究者**：直接读 3，再回看 1 的「一条数据如何流过系统」一节。

**想 fork 本仓库的开发者**：必须读 2 和 3。第二篇明确「上游 vs 本分叉」的边界，第三篇的六条原则会约束你的改动方向——比如你想加新 setup，必须能通过统计显著性纪律的检查（CI 不跨 0、尾部占比可接受）。

## 关键事实速查

如果只想记三个事实：

1. **两条独立管线**：`--auto` 全市场筛选（30 天周期）和 `--daily-action` 凸性 setup（T+5/T+10），只共享缓存数据。
2. **回测数据在 `data/paper_trading_backtest/`**：403 条记录、nav=2.10、realized_pnl=+110%。不是 `data/paper_trading/`（运行时实例，0 笔 EXIT）。
3. **OversoldBounce 默认暂停**：E[r]=+0.34% 表面为正，但 95% CI `[-3.15%, +3.83%]` 跨 0、尾部亏损占比（>10% 占 20%）比 BTST（11%）厚、仓位受限时有统计显著的 BTST 替代品。暂停依据是「无法证明赚钱」——「crisis 子样本亏钱」只是 n=21 的现象，不能作为独立决策依据。
