## v2.1.9 (2026-06-16) — AutoDev 数据正确性 campaign 1-6

### 主要变更
一轮系统性的数据正确性审计（AutoDev 驱动），修复 11 个真实 bug，记录 4 个需产品决策的 backlog。

**已修复 bug（BH-1 ~ BH-11）：**
- BH-1 (P2): `_portfolio_values` seed 行与首个 bar 共享 Date → 重复 Date index（engine + backend 同类 drain）。锚点改到前一自然日。
- BH-2 (P1): T+30 edge 旁"样本"数含未成熟 picks（survivorship/maturity bias）。新增 per-horizon 成熟样本计数，所有展示位置诚实化。→ R35
- BH-3 (P2): 组合胜率颜色阈值 0.45 与全局 0.50 不一致。对齐。
- BH-4 (P2): benchmark 迭代全部 tracking 而非当日 basket，Top-N trimming 打破 excess_return≡0。限制为 basket。
- BH-5 (P1): 连续推荐 streak 用自然日步进，每个周末断裂、清零 R4 加权。改为交易日步进。→ R36
- BH-6 (P1): R9 评分趋势箭头在 `previous_score=0.0`（归零）时误显"→"。增加 `change_pct is None` 守卫。
- BH-7 (P1): tracking lookback 锚定墙钟 `datetime.now()`，回填数据被静默丢出窗口。新增 `as_of` 参数。
- BH-8 (P1): re-run tracking 无条件覆盖已有 realized return，数据源返回更短序列时会退化已成熟记录。改为 merge（只采纳非 None）。
- BH-9 (P1): valuation transition-stage FCF 用 `(1+rate)^(year-3)` 而非累积复利，正增长期 FCF 非单调下降，低估 terminal value。改为 running product。
- BH-10 (P2): `build_front_door_verdict` 无条件列出"T+30 edge 转负"，BUY（edge>0）也误显示。条件化为 `t30_edge < 0`。
- BH-11 (P1): composite/conviction 排序无 secondary key，相同分 ticker 跨运行 Top-N 非确定。加确定性 tie-break。

**重构/清理：**
- 删除 4 个 unreferenced 根目录调试脚本（`debug_agent_facts.py` 等），规则已沉淀进 `src/agents/prompt_rules.py`。
- DRY `confidence_calibration` 的 win_rate/avg_return 重复计算（`_win_rate_or_none` / `_mean_or_none` helper）。

**新增 backlog（需产品决策，⏳）：**
- R37: 回测价格复权策略明确化（Tushare 不复权 + 无 corporate-action 处理）。
- R38: 回测交易日历用 A 股真实交易日（trade_cal）。
- R39: composite_score fallback domain mismatch（score_b → composite_score 绕过 penalties）。
- R40: macro_data look-ahead（fetch_macro_snapshot 无 trade_date 过滤，当前 informational-only）。

## v2.2.0 (2026-06-16) — AutoDev 用户授权 backlog 实施（campaign 7-9）

用户明确授权 R37/R38/R39 全部实施，按建议优先级 R37 → R39 → R38 执行。

### R37 (P1) ✅ — A 股价格改前复权（qfq）
`_fetch_tushare_ashare_prices_df` 从 `pro.daily`（不复权）切换到 `ts.pro_bar(adj="qfq")`。除权除息日（送股/分红/配股）不再产生假跳空缺口，回测收益/回撤/ATR/止损计算干净。复权只作用于价格水平，不改变收益结构。新增 `_cached_tushare_pro_bar_call` / `_call_tushare_pro_bar_api`（pro_bar 是顶层函数，复用缓存+重试机制）。

### R39 (P1) ✅ — composite_score fallback domain mismatch 修正
`rank_recommendations_by_investability` 的 fallback 此前把 `score_b`（[0,1]）直接赋给 `composite_score`（[-1,1]，含负 penalties），绕过调整。改为应用 0.9 保守折扣（score_b=0.55 → 0.495 < BUY 0.5 门控）+ `composite_verified=False` 标记。验证：正常流程 latent（composite 覆盖所有 ticker），仅 composite 异常时触发。

### R38 (P2) ✅ — 回测交易日历用 A 股真实交易日
`iter_backtest_dates` 从 `freq="B"`（周一-周五）改为优先用 `get_open_trade_dates`（trade_cal），排除春节/国庆等节假日，消除 phantom zero-return bar。空列表/异常时 fallback 到 `freq="B"`（保证无 token 仍可运行）。

### 路线图完成度
- 总体 39/40（R35-R39 ✅，R40 ⏳ latent 无消费方）
- 用户授权的三个高风险核心路径改进全部落地，经测试验证

## v2.2.1 (2026-06-16) — AutoDev campaign 10: ALPHA-006 + R40 + macro DRY

### ALPHA-006 (P2) ✅ — walk-forward drawdown_p10 小样本插值修正
`src/backtesting/early_runner_walk_forward.py` 的 `_distribution_p10` 此前用离散索引
`max(0, floor((N-1)*0.10))`，对 N<11 的样本索引恒为 0，p10 静默返回**最小值**（最差回撤）
而非真实 p10，系统性高估小窗口 walk-forward 报告的尾部风险。改为 `numpy.percentile` 线性插值，
与其它分析层一致，小样本也有可解释的尾部估计。该字段为 informational（不在 `_ranking_key`
或最优参数决策路径），不影响策略选择。新增 2 个回归测试（小样本插值 + 大样本尾部守卫）。

### R40 (P2) ✅ — macro_data as_of 点在时间过滤（消除 latent look-ahead）
`fetch_macro_snapshot` 新增可选 `as_of` 参数（YYYY-MM-DD / YYYYMMDD），通过 `_filter_df_as_of`
对所有指标（CPI/PPI/PMI/M2/社融/LPR）按 `month <= as_of_month` 过滤，消除 backtest/replay
场景读到模拟交易日后发布的宏观数据的 point-in-time look-ahead。`as_of=None`（默认）保持原
"取最新"行为（live 模式不变）。当前宏观 regime 仅作 `state.macro_context` informational label，
无决策消费方，且 `build_market_state` 不在 backtest 引擎路径，故此为前瞻性风险消除。3 个回归
测试（未来月排除 / 默认行为不变 / 早于全部数据返回 None）。

### 重构
- `macro_data.py`: 合并 6 个结构相同的 fetch helper（`_fetch_macro_cpi`/`_ppi`/`_pmi`/`_m2`/
  `_social_financing`/`_lpr_rate`）为单个 `_fetch_macro_frame(pro, api_name)`，约 -38 行重复
  代码，行为完全保持（46 个既有测试全绿）。

### 路线图完成度
- 总体 40/40（R35-R40 全部 ✅）
- 活跃 backlog 清零

## v2.2.2 (2026-06-16) — AutoDev campaign 11: 前门 BUY 门控成熟样本对齐（R35 一致性收口）

### 前门 BUY 门控用成熟 T+30 样本（R35 一致性 drain）✅
`build_front_door_verdict`（`--top-picks` 默认前门）的 BUY 门控此前用
`bucket_sample_count >= 20`，即桶内全部历史推荐数（含尚未满 30 天的未成熟记录）。
这与 R35 的展示诚实化目标矛盾：R35 让用户看到 T+30 数字背后有多少是成熟样本，
但 BUY 决策本身仍由可能以未成熟样本为主的桶背书，T+30 edge/胜率的真实可信度低于
门控暗示的水平。改为当 R35 `bucket_t30_mature_count` 字段存在时，BUY 门控要求
`bucket_t30_mature_count >= 20`；字段缺失（pre-R35 / 部分流水线）时回退到原
`bucket_sample_count >= 20`，保证既有数据不被静默破坏。decision_flow 路径此前已读
`bucket_t30_mature_count`，前门至此对齐。服务于"更高确信"目标。3 个回归测试
（成熟样本不足不 BUY / 足够仍 BUY / 缺字段回退原行为）。

### 审计结论
- Bug backlog 的 GAMMA-010（daily trade limit 用 amount 而非 cash）与 BETA-009
  （binding_constraint cash<min_lot 标签）经源码精读确认为 **by-design**：daily limit
  是相对 portfolio_nav 的敞口约束（非现金可用性约束，后者由 `cash_limit` 独立处理）；
  `min(constraints)` 选取 binding 约束在数学上正确。两者降级为已审计的 non-bug。

### 路线图完成度
- 总体 40/40（无新增 active item；本项为 R35 的逻辑一致性收口）
- 活跃 backlog 维持清零状态

## v2.2.3 (2026-06-16) — AutoDev campaign 13: GAMMA-007 embargo-bound prefetch

### GAMMA-007 (P2) ✅ — walk-forward prefetch embargo guard (latent look-ahead hardened)
`MarketDataLoader.prefetch_data` 此前无条件拉取到 `end_date`（1年窗口），依赖每个
walk-forward 调用方自行保证 `end_date` 是测试窗口的最后交易日而非整个回测末日，否则
会把窗口后的价格灌入特征缓存（latent point-in-time look-ahead）。

新增可选 `data_through` 参数：设置后 prefetch 把特征预热窗口夹到
`min(end_date, data_through)`，对 per-ticker 价格/指标/新闻 **和** benchmark 序列
全部生效，让 no-look-ahead 契约自执行而非依赖调用方约定。该参数已贯穿
`BacktestEngine.__init__`，walk-forward 调用方可传 `data_through=test_end` 启用。
`data_through=None`（默认）保持原单窗口行为。

审计确认当前调用方（`run_backtest_param_grid._build_engine` 每窗口 `end_date=test_end`）
已天然夹住迭代，故 bug 为 latent——本次为防御性硬化，把潜在风险变为不可能。
2 个回归测试（data_through 夹断 / 默认回退 end_date）。

### 新领域研究（option 3）：execution / paper_trading 域
对 `src/paper_trading/` 与 `src/execution/` 做了首轮结构性审计（此前 campaign 均未触
及）。结论：该域成熟度高——`run_paper_trading_session` 为 prepare→run→finalize 三段式，
backtest 引擎已有 checkpoint/resume（engine.py:932），finalization 经 helper 解耦。
**本轮无可执行的产品质量差距**（幂等性/恢复是值得后续深入的角度，但非低风险单切片）。
记录为 exhausted-domain-research，作为后续 campaign 的研究输入。

### 路线图完成度
- 总体 40/40（无新增 active item；GAMMA-007 为 backtest-engine 防御性硬化）
- bug backlog GAMMA-007 [DEFERRED] → [RESOLVED]
