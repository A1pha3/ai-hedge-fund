## v2.1.9 (2026-06-16) — AutoDev 数据正确性 campaign 1-6

### 主要变更
一轮系统性的数据正确性审计（AutoDev 驱动），修复 11 个真实 bug，记录 4 个需产品决策的 backlog。

**已修复 bug（BH-1 ~ BH-11）：**
- BH-1 (P2): `_portfolio_values` seed 行与首个 bar 共享 Date → 重复 Date index（engine + backend 同类 drain）。锚点改到前一自然日。
- BH-2 (P1): T+30 edge 旁"样本"数含未成熟 picks（survivorship/maturity bias）。新增 per-horizon 成熟样本计数，所有展示位置诚实化。→ R35
- BH-3 (P2): 组合胜率颜色阈值 0.45 与全局 0.50 不一致。对齐。
- BH-4 (P2): benchmark 迭代全部 tracking 而非当日 basket，Top-N trimming 打破 excess_return≡0。限制为 basket。
- BH-5 (P1): 连续推荐 streak 用自然日步进，每个周末断裂、清零 R4 加权。改为交易日步进。→ R36
- BH-6 (P1): R9 评分趋势箭头在 `previous_score=0.0`（归零）时误显"→"。增加 `change_pct is None` 守卫。
- BH-7 (P1): tracking lookback 锚定墙钟 `datetime.now()`，回填数据被静默丢出窗口。新增 `as_of` 参数。
- BH-8 (P1): re-run tracking 无条件覆盖已有 realized return，数据源返回更短序列时会退化已成熟记录。改为 merge（只采纳非 None）。
- BH-9 (P1): valuation transition-stage FCF 用 `(1+rate)^(year-3)` 而非累积复利，正增长期 FCF 非单调下降，低估 terminal value。改为 running product。
- BH-10 (P2): `build_front_door_verdict` 无条件列出"T+30 edge 转负"，BUY（edge>0）也误显示。条件化为 `t30_edge < 0`。
- BH-11 (P1): composite/conviction 排序无 secondary key，相同分 ticker 跨运行 Top-N 非确定。加确定性 tie-break。

**重构/清理：**
- 删除 4 个 unreferenced 根目录调试脚本（`debug_agent_facts.py` 等），规则已沉淀进 `src/agents/prompt_rules.py`。
- DRY `confidence_calibration` 的 win_rate/avg_return 重复计算（`_win_rate_or_none` / `_mean_or_none` helper）。

**新增 backlog（需产品决策，⏳）：**
- R37: 回测价格复权策略明确化（Tushare 不复权 + 无 corporate-action 处理）。
- R38: 回测交易日历用 A 股真实交易日（trade_cal）。
- R39: composite_score fallback domain mismatch（score_b → composite_score 绕过 penalties）。
- R40: macro_data look-ahead（fetch_macro_snapshot 无 trade_date 过滤，当前 informational-only）。

## v2.2.0 (2026-06-16) — AutoDev 用户授权 backlog 实施（campaign 7-9）

用户明确授权 R37/R38/R39 全部实施，按建议优先级 R37 → R39 → R38 执行。

### R37 (P1) ✅ — A 股价格改前复权（qfq）
`_fetch_tushare_ashare_prices_df` 从 `pro.daily`（不复权）切换到 `ts.pro_bar(adj="qfq")`。除权除息日（送股/分红/配股）不再产生假跳空缺口，回测收益/回撤/ATR/止损计算干净。复权只作用于价格水平，不改变收益结构。新增 `_cached_tushare_pro_bar_call` / `_call_tushare_pro_bar_api`（pro_bar 是顶层函数，复用缓存+重试机制）。

### R39 (P1) ✅ — composite_score fallback domain mismatch 修正
`rank_recommendations_by_investability` 的 fallback 此前把 `score_b`（[0,1]）直接赋给 `composite_score`（[-1,1]，含负 penalties），绕过调整。改为应用 0.9 保守折扣（score_b=0.55 → 0.495 < BUY 0.5 门控）+ `composite_verified=False` 标记。验证：正常流程 latent（composite 覆盖所有 ticker），仅 composite 异常时触发。

### R38 (P2) ✅ — 回测交易日历用 A 股真实交易日
`iter_backtest_dates` 从 `freq="B"`（周一-周五）改为优先用 `get_open_trade_dates`（trade_cal），排除春节/国庆等节假日，消除 phantom zero-return bar。空列表/异常时 fallback 到 `freq="B"`（保证无 token 仍可运行）。

### 路线图完成度
- 总体 39/40（R35-R39 ✅，R40 ⏳ latent 无消费方）
- 用户授权的三个高风险核心路径改进全部落地，经测试验证
