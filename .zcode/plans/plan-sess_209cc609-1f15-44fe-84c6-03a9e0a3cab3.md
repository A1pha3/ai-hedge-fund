## Phase 1：修复 event_sentiment 的 4 个反向 bug

### 第一性原理

新闻数据已参与选股决策（生产权重 0.20，非文档写的 0.05），但当前实现有 **4 个「方向性错误」**——不是"没用好"，而是"用反了"或"重复劳动"。修这 4 个 bug 不需要调权重、不需要改控制流，是纯粹的纠偏。

---

### Bug 1：双重计算 sentiment（重复劳动 + 信息浪费）

**现状**：
- 摄入层 `akshare_news_helpers.py:classify_news_sentiment` 用 ~150 个中文关键词（涨停/跌停/利好/利空/减持/预增...）给每条新闻打 `sentiment` 字段（positive/negative/neutral），**已存入快照**
- 评分层 `strategy_scorer_event_sentiment_helpers.py:_count_event_keyword_hits` 用 36 个词（18+18）**重新算一遍**，且词表更小、结果更差
- 摄入层的 `sentiment` 字段在评分路径中**从未被读取**（grep 确认）

**修复**：`_count_event_keyword_hits` 改为**优先用已有的 `item.sentiment` 字段**，仅当 sentiment=neutral 时再用关键词表做二次判定（捕捉摄入层大表遗漏的强事件词）。

**改动文件**：`src/screening/strategy_scorer_event_sentiment_helpers.py`
- `_count_event_keyword_hits` 函数（L384-388）：先读 `news_item.sentiment`
  - `"positive"` → 返回 `(2, 0)`（strength=2，足够触发 direction）
  - `"negative"` → 返回 `(0, 2)`
  - `"neutral"` 或 `None` → 走原有 36 词匹配（保留兜底）

---

### Bug 2：关键词表太小，A 股事件词严重缺失

**现状**：评分用的 `POSITIVE_NEWS_KEYWORDS`（18 词）和 `NEGATIVE_NEWS_KEYWORDS`（18 词）偏向美股财报语义（beat/miss/upgrade/downgrade），A 股核心事件词几乎缺失：无"涨停/跌停/利好/利空/预增/预减/增持/减持完成/被立案/问询函/股权激励/重组/复牌"等。

**修复**：扩充 `strategy_scorer_utils.py` 的两张词表，从 `akshare_news_helpers.py:classify_news_sentiment` 的大表（~150 词）中精选 A 股高频事件词，使**兜底路径（Bug 1 的 neutral 二次判定）也有足够覆盖度**。

**改动文件**：`src/screening/strategy_scorer_utils.py:53-92`
- `POSITIVE_NEWS_KEYWORDS`：18 → ~45 词（新增涨停/大涨/暴涨/利好/突破/创新高/签约/获批/扭亏/新高/强势/反弹/爆发/景气/提价/提升/改善/翻倍/增持/预增/业绩预增 等）
- `NEGATIVE_NEWS_KEYWORDS`：18 → ~45 词（新增跌停/大跌/暴跌/利空/问询函/关注函/立案/停牌/退市/爆雷/债务/萎缩/破发/破位/新低/跳水/杀跌/闪崩/抛售/出逃/预降/预减/业绩变脸/业绩不及预期 等）

**原则**：只增不减，不删除任何现有词；新增词从 `classify_news_sentiment` 已验证的大表中选取。

---

### Bug 3：多源报道被当噪声丢弃（共识信号反向处理）

**现状**：`akshare_news_helpers.py:deduplicate_news`（L255-292）用 Jaccard 标题相似度 ≥0.5 判定为重复并**直接删除**。但"同一事件被多个来源报道"恰恰是**信号强化**（共识确认），不是噪声。日志里经常出现 `新闻去重：移除 N 篇重复报道`——这些被删的报道本该提升 confidence。

**修复**：不改变去重行为（去重仍需要——避免完全相同的文章重复计数），但保留 cluster 元数据，让评分层能利用"多源共振"信号。

**方案**：
- `deduplicate_news` 返回结果不变（去重后的列表），但每条 article 附加 `dedup_cluster_size` 字段（该文章代表的"同事件报道簇"大小）。保留第一条，记录该簇有多少篇被去重。
- `CompanyNews` 模型新增可选字段 `dedup_cluster_size: int = 1`
- 评分层 `_score_news_article`（L199-206）：`confidence` 按 `cluster_size` 加成：`confidence *= min(1.5, 1.0 + 0.1 * (cluster_size - 1))`，即 2 篇同事件 → ×1.1，5 篇 → ×1.4，上限 ×1.5。这模拟"多源共识 → 更可信"。

**改动文件**：
- `src/data/models.py`：`CompanyNews` 加 `dedup_cluster_size: int = 1`
- `src/tools/akshare_news_helpers.py`：`deduplicate_news` 内部统计 cluster_size 并写入保留文章
- `src/screening/strategy_scorer_event_sentiment_helpers.py`：`_score_news_article` 读取 cluster_size 做 confidence 加成

---

### Bug 4：insider_conviction 死区稀释有效子因子

**现状**：`_score_insider_conviction`（L292）：`completeness = 0.0 if (direction == 0 and confidence == 0.0) else 1.0`。但 `_resolve_insider_conviction_direction_and_confidence`（L303-304）：direction=0 的死区是 `score ∈ [0.4, 0.6]`，此时 `confidence = abs(score - 0.5) * 200`，只要 score ≠ 0.5，confidence 就 > 0。

所以 `direction=0, confidence=10`（score=0.45）时，completeness 仍 = 1.0 → 这个**无方向信号的子因子占据了 25% 权重**，在 `aggregate_sub_factors` 里稀释了 news_sentiment 的有效权重。

注释 L288-291 明确说"When direction=0 and confidence=0 the sub-factor carries no information, so completeness=0.0"——意图是对的，但条件写窄了，应该是 `direction == 0`（无方向 = 无信息）。

**修复**：L292 改为 `completeness = 0.0 if direction == 0 else 1.0`。direction=0 的内部人交易不携带方向性信息，不应占据权重。

**改动文件**：`src/screening/strategy_scorer_event_sentiment_helpers.py:292`

---

### 不改什么

- **不改策略权重**（0.20 生产权重不动）——修完 bug 后信号质量自然提升，权重调优留给 Phase 2 数据驱动决定
- **不改时间衰减系数**（EVENT_DECAY_LAMBDA=0.35、_event_weight_multiplier 阶跃表）——这些经过 R119/GAMMA 修复，不动
- **不改 completeness 归一化逻辑**——这是正确的降级机制
- **不改候选选择逻辑**（EVENT_SENTIMENT_MAX_CANDIDATES）——留给 Phase 2
- **不改 content 截断**（200 字）——放开截断需要爬全文，是另一个量级的工作
