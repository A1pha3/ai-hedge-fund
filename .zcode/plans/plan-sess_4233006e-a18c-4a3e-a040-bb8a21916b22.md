## 改动范围：3 个文件 + 2 个测试文件

### 目标
1. **方案 A**：给价格主链 `load_prices_with_fallback` 加 Tushare 第 3 层兜底（AKShare → 腾讯 → Tushare），复用已实现的 `TushareDataSource.get_prices`。
2. **日志准确性改造**：修正价格链和资金流链的误导性/丢失性日志。

---

### 文件 1：`src/tools/akshare_price_helpers.py` — 核心改动

**`load_prices_with_fallback` 函数重写**（`:178-229`）：

- **新增参数** `fetch_prices_from_tushare_fn: Callable[..., list[Price]] | None = None`（可选，None 时跳过 Tushare 层，保持向后兼容）
- **改为三层回退**：AKShare → 腾讯 → Tushare，每层捕获各自的 error。
- **日志修正**：
  - AKShare 层失败：`"[价格链 1/3] AKShare %s 失败 (%s): %s, 尝试腾讯接口"` — 保留真实 error message（不再只打 `error_name`），修正"代理问题非bug"的武断归因。
  - 腾讯层失败：新增 `"[价格链 2/3] 腾讯接口 %s 失败 (%s): %s, 尝试 Tushare"`（此前完全没有腾讯层日志，是最大的日志盲区）。
  - Tushare 层失败：新增 `"[价格链 3/3] Tushare %s 失败 (%s): %s"`。
  - 三层全空时：最终 raise 的 message 列出三层各自的真实 error。
- **网络错误去重计数器**：`_net_count` 的逻辑保留（批量场景防止刷屏），但去掉"代理问题, 非代码bug"的归因语句——改为中性描述"网络错误累计 N 次, 后续静默"，由 operator 自行判断。

### 文件 2：`src/tools/akshare_api.py` — 传入 Tushare fetcher

**`get_prices` 函数**（`:326-333`）：在 `load_prices_with_fallback` 调用处新增 `fetch_prices_from_tushare_fn=_get_prices_from_tushare`。

**新增辅助函数 `_get_prices_from_tushare`**（在 `_get_prices_from_tencent` 附近，约 `:257`）：包装 `TushareDataSource.get_prices`，把 `DataSourceError` 转成 `AShareDataError`，lazy import 避免循环依赖。返回 `list[Price]`（空时返回 `[]`，空列表不触发 fallback——与 AKShare 层 `if akshare_prices:` 语义一致，用 `if prices:` 守卫）。

### 文件 3：`src/tools/fund_flow.py` — 资金流日志修正

**`fetch_individual_fund_flow` 函数**（`:49-67`）：
- 修正双源空时的误导日志。当前 `[%s] 资金流双源均空 — tushare+akshare 均无数据` 在 akshare 是 SSLError（网络失败）时是**错误的**——akshare 不是"无数据"而是"抓取失败"。
- 改为记录每源的真实结果（空 vs 异常）：在循环里追踪每源的失败原因（None=空，或异常对象），最终消息按真实情况组合：`"[资金流] %s 双源均失败 — tushare: <空/异常原因>; akshare: <空/异常原因>"`。
- 去掉"可能是北交所/新股等数据源未覆盖"的硬编码推测（002281 是正常深市主板，这个推测本身就是错的），改为中性陈述。

### 文件 4 & 5：测试更新

**`tests/test_ns17_akshare_price_helpers_observability.py`**（`:118-129`）：
- `load_prices_with_fallback` 调用新增 `fetch_prices_from_tushare_fn` 参数（传一个 mock 或 None）。
- 可选：新增测试验证腾讯层失败时发 warning（补上当前的日志盲区测试）。

**`tests/test_r20_4_regressions.py`**（`:341-352`）：
- `load_prices_with_fallback` 调用新增 `fetch_prices_from_tushare_fn` 参数。
- 该测试验证"两层都失败时 error message 含两者"——现在改为三层，message 须含三层（或保持只验证 AKShare + 腾讯两层 error 在 message 中，Tushare 传 None 跳过）。

### 向后兼容性
- `fetch_prices_from_tushare_fn` 默认 None → 老调用方（如有）不传也不会 break，只是没有第 3 层兜底。
- 现有两个测试只需补一个参数即可通过。

### 不改的部分
- 不动 `execute_robust_price_request`（已是 4 层链，有完整日志，不是本次问题）。
- 不动 `akshare_fund_flow.py` 的 SSLError 日志（它本身已区分网络错误去重，准确；问题在 `fund_flow.py` 的聚合消息）。
- 不加 `moneyflow_dc` 资金流第 3 源（用户确认只做方案 A）。
