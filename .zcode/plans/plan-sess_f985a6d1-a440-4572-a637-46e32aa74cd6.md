## 修复方案：5 个问题，第一性原理驱动

### Bug 1：因子瀑布的 `stab`/`other` 伪分解（核心）

**根因**：`compute_score_decomposition`（`signal_fusion.py:629`）把 `stability_bonus`(0-10量纲) 和 `attention_composite` 当成 `score_b`(±1量纲) 的加性分量。但两者**从未被加进 score_b**——`compute_score_b`(:359-385) 只包含 base contributions + consensus_bonus。此外还有**存盘/打印不一致**：存盘的 decomposition 在 enrichment 前注入（stability_bonus=0），打印时重新计算（stability_bonus=10），造成 JSON other=0 但日志 other=-10。

**修复**（`src/screening/signal_fusion.py:628-630`）：
```python
# 旧：components_sum = base_sum + attention + stability_bonus + consensus_bonus
# 新：只有 base + consensus 是 score_b 的真实加性分量
components_sum = base_sum + consensus_bonus
```
- `attention` 和 `stability_bonus` 仍作为独立元数据字段返回（保留在 dict 里），但**不参与求和**，因此 `other_adjustments` 现在是真实的 score_b - (base+consensus) 残差（通常 ≈ 0，仅 clamp 边界有微小残差）。

**修复存盘/打印不一致**（`src/main.py:807`）：
- 把 `_inject_score_decomposition` 调用**移到** `enrich_recommendations_with_history`（:820）**之后**。
- 修改 `_inject_score_decomposition`（:690）签名，接受 `consecutive_lookup` 并传给 `compute_score_decomposition`，让存盘和打印用同一份数据。

**显示调整**（`src/main.py:1802-1820` `_print_score_waterfall`）：
- `att` 和 `stab` 行改为标注 `(non-additive context)` 而非参与瀑布求和。
- `other` 行标签从 `(market-state + residual)` 改为 `(clamp residual)`——反映其真实含义。

### Bug 2：行业轮动只基于 Top-10（用 fused 全市场）

**修复**（`src/main.py:834-836`）：用 `fused`（~300只全市场快筛，line 778）替代 `top_results_serializable`（10只）：
```python
fused_serializable = [item.model_dump(mode="json") for item in fused]
industry_signals = calculate_industry_rotation(
    recommendations=fused_serializable,  # 全市场快筛 ~300只
    trade_date=trade_date,
)
```
全市场横截面让强弱行业真正分化（min_candidates=2 过滤后仍有多行业存活），消除"强弱行业重复"问题。

### Bug 3：排序键不透明（加 Composite 列）

**修复**（`src/main.py:1940-1949` `_build_auto_screening_table_row` + headers）：
- 在 "Score B" 列后新增 "Composite" 列，显示 `composite_score`（主排序键），让排序自解释。
- composite_score 在 `investability.py:450` 写入 rec dict。print 路径 top_results 是 FusedScore 对象无此字段，实现时通过额外传入 `composite_by_ticker` 映射（从 ranked_pool 的 rec dict 提取）来填充该列，不污染 FusedScore 模型。

### Bug 4：event_sentiment 空信号稀释权重（定点修复 completeness）

**根因**：`event_freshness`（:340-346）和 `insider_conviction`（:288-294）子因子在 direction=0/confidence=0 时仍返回 completeness=1.0（`_make_sub_factor` 默认值），使 event_sentiment 整体 completeness=1.0，占用 ~10% 权重稀释 T/MR/F。

**修复**（`src/screening/strategy_scorer_event_sentiment_helpers.py`）：
- `_build_event_freshness_factor`（:340）：当 `snapshot.freshness_weight <= 0` 时，`completeness=0.0`。
- `_score_insider_conviction`（:288）：当 `direction == 0 and confidence == 0` 时，`completeness=0.0`。
- `_resolve_news_sentiment_signal`（:177）已基于 informative_count 正确处理，不改。

这样"完整但无信息"的信号 completeness→0，被 `_normalize_active_weights` 正确剔除并重归一化，T/MR/F 权重恢复。

### Bug 5：`_print_top_score_enhancements` 的 top_n 死参数（次要）

**修复**（`src/main.py:1541`）：把 `recs[:5]` 改为 `recs[:top_n]`。
