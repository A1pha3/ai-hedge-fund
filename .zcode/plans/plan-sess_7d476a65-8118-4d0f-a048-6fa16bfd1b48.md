## 目标

从第一性原理消除整个 bug 类：**一个 token 加载器，一个 `get_pro()`，全代码库不再调用 `ts.set_token()`。**

## 根因回顾

当前 3 种认证模式 + 9 个重复的 `_load_token()` 实现，产生两类 bug：
1. `ts.set_token(None)` 静默写 `NaN` 到 `~/tk.csv` → 所有依赖 token 文件的路径失效（今天的 `industry_index_cache` 失败）
2. `backfill_industry_index.py` 用 Pattern C（只设 env 不写文件），当 `~/tk.csv` 被别的脚本写脏就失效

## 设计原则

- **单一数据源**：一个 `get_tushare_token()` 函数，env 优先、`.env` 文件兜底，带 None 守卫
- **无副作用认证**：所有 `ts.pro_api(token=...)` 直接传参，**永不调用 `ts.set_token()`**（消除文件耦合）
- **单一入口**：所有代码通过 `_get_pro()` 获取 pro 实例（已有线程安全单例）
- **渐进迁移**：scripts 保留各自 `pro = _get_pro()` 调用方式，只是 import 来源统一

## 改动清单

### 1. 新增 token 加载器（src/tools/tushare_api.py）

在 `_get_pro()` 旁新增 `get_tushare_token()` 公开函数：
```python
def get_tushare_token() -> str:
    """返回 Tushare token (env 优先, .env 文件兜底). 永远不返回 None."""
    token = os.environ.get("TUSHARE_TOKEN", "").strip()
    if token:
        return token
    # .env 兜底 (不依赖 python-dotenv, 避免循环导入)
    env_path = Path(__file__).resolve().parents[2] / ".env"
    if env_path.exists():
        for line in env_path.read_text(encoding="utf-8").splitlines():
            if line.startswith("TUSHARE_TOKEN="):
                return line.split("=", 1)[1].strip().strip("'\"")
    return ""
```

修改 `_get_pro()` 第 332 行：
```python
# before:  token = os.environ.get("TUSHARE_TOKEN")
# after:   token = get_tushare_token()
```
这样 `_get_pro()` 获得 `.env` 兜底能力，同时保持 Pattern B（直接传参，不写文件）。
