## 目标

控制台输出做到"一眼看清发生了什么"：rich 状态行展示进度，关键 INFO 摘要展示结果。冗余/内部日志降级 DEBUG（控制台不见，文件保留）。

## 改动清单

### 1. 控制台日志格式精简 (`src/utils/logging.py`)

控制台 formatter 从 `时间 | 级别 | 模块:函数:行号 | 消息` 改为 `时间 级别 消息`：

```python
# console formatter
"console": "%(asctime)s | %(levelname)-8s | %(message)s"
# file formatter (不变 — 保留完整调试信息)
"file": "%(asctime)s | %(levelname)-8s | %(name)s:%(funcName)s:%(lineno)d | %(message)s"
```

### 2. 冗余日志降级 DEBUG (`src/main.py`)

这些 `logger.info` 与 rich 状态行重复或属于内部细节，降为 `debug`：

| 行 | 当前 | 改为 | 理由 |
|---|---|---|---|
| 784 | `[Auto] P0-1 BatchDataFetcher: use_batch=...` | `debug` | 内部配置 |
| 792 | `[Auto] Step 1/4: 全市场快筛` | `debug` | rich 状态行已显示 |
| 810 | `[Auto] Scoring feature refresh status=...` | `debug` | 内部状态 |
| 817 | `[Auto] Step 2/4: 四策略评分` | `debug` | rich 状态行已显示 |
| 829 | `[Auto] Step 3/4: 信号融合` | `debug` | rich 状态行已显示 |
| 834 | `[Auto] Step 4/4: 输出 Top 10` | `debug` | rich 状态行已显示 |
| 866 | `[Auto] score_decomposition injected...` | `debug` | 内部细节 |

保留为 INFO 的（控制台可见）：
- `[Auto] Layer A 候选池: 300 只` — 关键结果
- `score_batch completed: 300, 123 heavy, 1.27s` — 关键性能指标
- `[cache_refresh] 注入涨停` / `资金流统计` — 关键结果

### 3. 新增摘要行 (`src/main.py`)

在关键节点用 `logger.info` 输出一行精简摘要：

**cache_refresh 完成后**（`_refresh_daily_action_caches_for_auto` 末尾）：

```python
    "[Auto] 缓存刷新: 行业 %d 行业/%d行%s, 涨停注入 %d, 资金流 %d跳过/%d新增/%d停牌/%d空",
    ...stats fields...
)
```

### 4. 精简 cache_refresh 子模块日志 (`src/screening/offensive/cache_refresh.py`)

- `注入 1 只当日涨停股 (pct>=9.5%): ['301583']` → 保留 INFO，精简格式
- `完成: 31 行业, 总 48949 行` (from `backfill_industry_index.py`) → 降级 DEBUG（摘要行已包含）

### 5. 精简 strategy_scorer 日志

`score_batch completed: 300 candidates, 123 heavy-scored, concurrency=4, 1.27s` → 精简为 `评分: 300 只 · 123 heavy · 1.27s`
