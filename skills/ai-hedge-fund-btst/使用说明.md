# ai-hedge-fund-btst 使用说明

## 1. 这是做什么的

这个 skill 用于在当前仓库里生成 BTST 次日文档包。

默认流程只有 6 步：

1. 解析信号日和次日交易日
2. 运行规则版 BTST 报告
3. 运行 short_trade_only 的多智能体 paper trading 流程
4. 刷新或读取 early-runner 产物
5. 读取当次产物
6. 用统一脚本写出最终中文 Markdown

## 2. 默认会产出什么

默认会生成整套 5 份主文件，文件名使用信号日 YYYYMMDD：

1. BTST-YYYYMMDD.md
2. BTST-LLM-YYYYMMDD.md
3. YYYYMMDD-两套交易计划通俗说明.md
4. YYYYMMDD-两套交易计划论坛短版.md
5. BTST-YYYYMMDD-EXEC-CHECKLIST.md

如果启用了方案 A 的 early-runner 扩展，还会额外生成 2 份补充文档：

6. BTST-YYYYMMDD-EARLY-WARNING.md
7. BTST-YYYYMMDD-EARLY-WARNING-CARD.md

除非你明确缩小范围，否则只提“交易计划”或“通俗说明”，也会按整套输出处理。

## 3. 目录和日期规则

- 默认目录是 outputs/YYYYMM/
- 这里的 YYYYMM 来自信号日，不是目标交易日
- 方案 A 观察期建议目录是 outputs/YYYYMM/YYYYMMDD_scheme_a/
- 如果不指定日期，skill 会自动寻找“最新一个已有收盘数据的开市日”，再计算真实的下一交易日
- 如果指定了信号日，就按该日期执行；如果当天收盘数据不存在，会直接停止，不会硬编文档

开始执行前，会先问一句：

是否保存到默认目录 outputs/YYYYMM/？

你可以这样回答：

- yes
- 保存到默认目录
- 不用默认目录，保存到 /你的/自定义/路径

如果同时给了默认和自定义路径，以自定义路径为准。

## 4. 推荐写法

最稳的一句：

生成 X 月 X 日的 BTST 全套文档，使用 X 月 X 日收盘数据，保存到默认目录。

也可以用这些写法：

- 生成明天的 BTST 全套文档，保存到默认目录。
- 生成 X 月 X 日的 BTST 交易计划，使用 X 月 X 日收盘数据。
- 生成 X 月 X 日的 BTST 通俗说明，保存到 /你的/输出目录。

## 5. 文档是怎么写出来的

这个 skill 现在优先通过统一脚本生成文档包：

```bash
uv run python scripts/generate_btst_doc_bundle.py \
  --signal-date YYYYMMDD \
  --output-dir outputs/YYYYMM/YYYYMMDD_scheme_a \
  --no-refresh-early-runner
```

这个 skill 不只是跑脚本，还会基于当次真实产物组织正式 BTST、early-runner 和交集票高亮结果。

当前约定是：

- 规则报告和多智能体流程可以自动执行
- 最终文档不能靠固定模板硬拼
- 通俗说明和论坛短版要根据当天产物重写，不能日复一日复读同一段话
- 市场背景必须来自当前产物和可观察上下文，不能写成和当天无关的固定盘前点评
- 如果 early-runner 与正式 BTST 同时命中同一只票，文档会增加“交集票高亮”
- 如果 early-runner 没有当日板，文档必须显式写出 `stale_fallback` 或 `unavailable`，不能伪造当日观察票

## 6. 如果 Copilot 命令行找不到这个 skill

有些环境不会自动扫描仓库里的 skills 目录，而只扫描全局目录。

这时可以运行：

```bash
./skills/ai-hedge-fund-btst/scripts/install_symlink.sh
```

默认会链接到：

```bash
~/.copilot/skills/ai-hedge-fund-btst
```

如果目标已存在同名目录或链接：

```bash
./skills/ai-hedge-fund-btst/scripts/install_symlink.sh --force
```

如果你想指定别的 skills 根目录：

```bash
./skills/ai-hedge-fund-btst/scripts/install_symlink.sh --target-root /你的/skills/目录
```

## 7. 常见问题

### Q1：为什么文件名用信号日，不用目标交易日？

因为这套文档的判断依据来自信号日收盘后的数据，当前命名约定也统一使用信号日。

### Q2：为什么有时只写“交易计划”，还是会生成 5 份文件？

因为这个 skill 的默认行为就是输出整套文档，除非你明确要求缩小范围。

### Q3：如果脚本过程中需要交互怎么办？

代理会继续询问所需输入，然后接着跑，不会静默跳过。

### Q4：如果当天数据还没准备好怎么办？

会先检查收盘数据是否可用；如果不可用，会直接停止并说明原因。

### Q5：为什么现在推荐先输出到 `scheme_a` 目录？

因为 early-runner 还处在观察期。先输出到 `outputs/YYYYMM/YYYYMMDD_scheme_a/`，可以避免把试运行结果直接混进正式归档目录。

### Q6：什么情况下可以切回正式目录？

建议满足下面条件再切回：

- 连续 10 到 15 个交易日都能稳定生成文档
- 最近连续 3 个交易日 `early_runner_status=exact`
- 最近连续 3 个交易日 `early_runner_latest_trade_date == signal_date`
- 已经开始稳定出现基于当日板的交集票高亮

## 8. 一句话版本

观察期里，平时直接用这句就够了：

生成 X 月 X 日的 BTST 全套文档，使用 X 月 X 日收盘数据，保存到 outputs/YYYYMM/YYYYMMDD_scheme_a/。
